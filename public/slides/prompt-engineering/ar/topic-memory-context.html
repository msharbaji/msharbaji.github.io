الذاكرة وإدارة السياق

امنح وكلاء الذكاء الاصطناعي ذاكرة دائمة — من مخازن المحادثات إلى قواعد البيانات المتجهية وأنظمة الذاكرة الهرمية.

1

مشكلة الذاكرة

في كل مرة تستدعي نموذجاً لغوياً كبيراً، يبدأ من الصفر تماماً. ليس لديه أي ذاكرة لما حدث من قبل. هذا قيد معماري جوهري — والتحدي الأكبر عند بناء وكلاء يحتاجون إلى الحفاظ على الاستمرارية.

المشكلات الأساسية واضحة:

  • النماذج اللغوية عديمة الحالة — كل استدعاء API يبدأ من الصفر
  • نوافذ السياق محدودة — لا يمكنك إرسال التاريخ بالكامل للأبد
  • المستخدمون يتوقعون الاستمرارية — "تذكّر ما أخبرتك به بالأمس"
  • الوكلاء يحتاجون السياق السابق — القرارات تعتمد على التفاعلات السابقة
⚡

الذاكرة قصيرة المدى

المحادثة الحالية — الرسائل في نافذة السياق النشطة. تُفقد عند انتهاء الجلسة.

💾

الذاكرة طويلة المدى

تُحفظ عبر الجلسات — تُخزَّن في قواعد بيانات أو ملفات أو مخازن متجهية. تنجو من إعادة التشغيل.

الذاكرة العرضية

أحداث وتجارب محددة — "أبلغ المستخدم عن خطأ يوم الثلاثاء" أو "نشرنا الإصدار 2.1 الأسبوع الماضي."

📚

الذاكرة الدلالية

معارف وحقائق عامة — "المستخدم يفضل بايثون" أو "المشروع يستخدم PostgreSQL."

💡

تشبيه: فقدان الذاكرة اللحظي

تخيّل نموذجاً لغوياً بدون ذاكرة كشخص مصاب بفقدان الذاكرة — بارع في اللحظة، لكنه لا يتذكر محادثة الأمس. في كل تفاعل، عليك أن تعيد تقديم نفسك وتشرح السياق من جديد. أنظمة الذاكرة هي الطريقة التي نعالج بها فقدان الذاكرة هذا.

أنواع الذاكرة

هناك أربعة أنماط أساسية للذاكرة (استناداً إلى إطار عمل Lilian Weng). كل نمط يقدم مقايضة مختلفة بين الاكتمال وكفاءة التوكنات والصلة.

📋

ذاكرة المخزن المؤقت

تخزين تاريخ المحادثة بالكامل. بسيطة ومكتملة، لكنها تنمو بلا حدود وتتجاوز نافذة السياق في النهاية.

🪟

ذاكرة النافذة

الاحتفاظ فقط بآخر N من الأدوار. تحدّ استخدام التوكنات لكنها تفقد السياق الأقدم — ينسى الوكيل الأجزاء الأولى من المحادثة.

📝

ذاكرة الملخص

استخدام النموذج اللغوي نفسه لتلخيص الرسائل القديمة. تحافظ على المعلومات الأساسية بتوكنات أقل، لكنها تفقد التفاصيل بطبيعتها.

🔍

الذاكرة المتجهية

تحويل الرسائل إلى متجهات واسترجاع الرسائل ذات الصلة فقط عبر البحث بالتشابه. تتوسع لتاريخات ضخمة.

ذاكرة المخزن المؤقت — الأسلوب الأبسط. خزّن كل شيء:

class BufferMemory:
    def __init__(self):
        self.messages = []

    def add(self, role: str, content: str):
        self.messages.append({"role": role, "content": content})

    def get_context(self) -> list[dict]:
        return self.messages.copy()

class SummaryMemory:
    def __init__(self, client):
        self.client = client
        self.summary = ""
        self.recent = []  # Last 3 turns

    def add(self, role: str, content: str):
        self.recent.append({"role": role, "content": content})
        if len(self.recent) > 6:  # 3 turns = 6 messages
            self._compress()

    def _compress(self):
        old = self.recent[:4]
        self.recent = self.recent[4:]
        # Ask LLM to update summary
        resp = self.client.messages.create(
            model="claude-haiku-4-5-20251001",
            max_tokens=300,
            messages=[{
                "role": "user",
                "content": f"Current summary:\n{self.summary}\n\n"
                           f"New messages:\n{old}\n\n"
                           "Update the summary concisely."
            }]
        )
        self.summary = resp.content[0].text

استراتيجيات نافذة السياق

الاستفادة القصوى من نافذة سياق محدودة أمر بالغ الأهمية. إليك الاستراتيجيات الرئيسية، من الأبسط إلى الأكثر تعقيداً:

الاستراتيجيةكيف تعملالمقايضة
الاقتطاع حذف الرسائل الأقدم عند امتلاء النافذة بسيط لكنه يفقد السياق المبكر بالكامل
النافذة المنزلقة الاحتفاظ فقط بآخر N توكن من المحادثة تكلفة متوقعة، لكن بدون استرجاع طويل المدى
التلخيص ضغط التاريخ في ملخص متجدد يحتفظ بالمعلومات الأساسية، لكنه يفقد التفاصيل
RAG استرجاع السياق ذي الصلة فقط من مخزن متجهي يتوسع جيداً، لكن جودة الاسترجاع حاسمة
الهرمي دمج الملخص + الرسائل الأخيرة + السياق المسترجع أفضل جودة، لكنه الأكثر تعقيداً في التنفيذ

نافذة السياق المصممة جيداً تخصص مساحة لكل مكون. إليك تخطيطاً نموذجياً:

⚠️

لا تملأ النافذة بالكامل

لا تملأ نافذة السياق بالكامل أبداً — اترك دائماً مساحة لرد النموذج. قاعدة جيدة: استخدم 75% كحد أقصى للإدخال. إذا ملأت النافذة بالكامل، فإن النموذج إما يقتطع رده أو ينتج مخرجات متدنية الجودة.

الذاكرة المتجهية

الذاكرة المتجهية تستخدم التضمينات (Embeddings) لتحويل النص إلى متجهات رقمية، ثم تسترجع الذكريات ذات الصلة عبر البحث بالتشابه. التدفق هو: نص → نموذج تضمين → متجه → تخزين في قاعدة بيانات → استعلام بالتشابه.

هذا النهج يحاكي طريقة عمل الذاكرة البشرية — أنت لا تتذكر كل محادثة أجريتها في حياتك. بدلاً من ذلك، السياق الحالي يحفز استرجاع الذكريات ذات الصلة.

Python 
import chromadb

client = chromadb.Client()
collection = client.create_collection("memories")

# Store a memory
collection.add(
    documents=["User prefers Python over JavaScript"],
    ids=["mem_001"],
    metadatas=[{"type": "preference", "date": "2025-01-15"}]
)

# Retrieve relevant memories
results = collection.query(
    query_texts=["What programming language should I recommend?"],
    n_results=3
)

# Inject into prompt
memories = "\n".join(results["documents"][0])
system = f"User context:\n{memories}\n\nBe helpful."

لماذا الذاكرة المتجهية مهمة

الذاكرة المتجهية هي أقرب شيء لطريقة تذكر البشر — أنت لا تتذكر كل شيء، فقط ما يتصل بالموقف الحالي. مخزن متجهي مضبوط جيداً يمكنه البحث عبر آلاف التفاعلات السابقة واستخراج السياق الذي يحتاجه الوكيل بالضبط.

الأنماط المتقدمة

أنظمة الذاكرة في بيئات الإنتاج تجمع عادة بين أنواع متعددة من الذاكرة في بنية هرمية. هذا يمنحك أفضل ما في كل العوالم — السياق الأخير مكتمل، والسياق الأقدم ملخص، وكل شيء آخر يمكن استرجاعه عبر البحث الدلالي.

Python 
class HierarchicalMemory:
    """Combines buffer + summary + vector memory."""

    def __init__(self, client, vector_store):
        self.client = client
        self.vector_store = vector_store
        self.recent = []          # Last 5 turns (buffer)
        self.summary = ""         # Rolling summary

    def add(self, role: str, content: str):
        self.recent.append({"role": role, "content": content})
        # Also store in vector DB for long-term retrieval
        self.vector_store.add(
            documents=[content],
            ids=[f"msg_{len(self.recent)}"],
            metadatas=[{"role": role, "type": "conversation"}]
        )
        if len(self.recent) > 10:
            self._compress()

    def get_context(self, current_query: str) -> str:
        # 1. Retrieve relevant past memories
        retrieved = self.vector_store.query(
            query_texts=[current_query], n_results=5
        )
        relevant = "\n".join(retrieved["documents"][0])

        # 2. Assemble hierarchical context
        return (
            f"## Conversation Summary\n{self.summary}\n\n"
            f"## Relevant Past Context\n{relevant}\n\n"
            f"## Recent Messages\n{self.recent}"
        )

    def _compress(self):
        old = self.recent[:6]
        self.recent = self.recent[6:]
        resp = self.client.messages.create(
            model="claude-haiku-4-5-20251001",
            max_tokens=300,
            messages=[{
                "role": "user",
                "content": f"Current summary:\n{self.summary}\n\n"
                           f"New messages:\n{old}\n\n"
                           "Update the summary concisely."
            }]
        )
        self.summary = resp.content[0].text

6

أفضل ممارسات التنفيذ

بناء أنظمة الذاكرة هو المكان الذي تلتقي فيه النظرية بالإنتاج. هذه هي الدروس الأكثر أهمية:

📈

ابدأ بالمخزن المؤقت، توسّع للمتجهات

ذاكرة المخزن المؤقت البسيطة تعمل للمحادثات القصيرة. لا تبالغ في الهندسة — أضف الذاكرة المتجهية فقط عندما تحتاج فعلاً للتعامل مع تواريخ طويلة أو الاسترجاع عبر الجلسات.

🧱

افصل بين أنواع الذاكرة

اجعل الذاكرة قصيرة المدى وطويلة المدى والعاملة أنظمة مستقلة. خلطها ينتج كوداً معقداً يصعب تنقيحه وضبطه.

🎯

حدد ميزانيات التوكنات دائماً

خصص ميزانيات توكنات ثابتة لكل مكون ذاكرة — مثلاً 500 توكن للملخص، 1000 للسياق المسترجع، 2000 للرسائل الأخيرة.

🧪

اختبر جودة الاسترجاع

الاسترجاع السيء أسوأ من عدم وجود ذاكرة — السياق غير ذي الصلة يربك النموذج. تحقق من أن البحث المتجهي يعيد نتائج مفيدة فعلاً.

✓

اختبر فهمك

اختبار سريع — 3 أسئلة

1. لماذا لا تستطيع النماذج اللغوية تذكر المحادثات السابقة بشكل افتراضي؟

2. متى تختار ذاكرة الملخص بدلاً من ذاكرة المخزن المؤقت؟

3. ما الميزة الرئيسية للذاكرة المتجهية؟

ملخص الموضوع 17

إليك ما تعلمته:

النماذج اللغوية عديمة الحالة — يجب إدارة الذاكرة بشكل صريح. أنواع الذاكرة الأربعة الأساسية هي المخزن المؤقت (التاريخ الكامل)، والنافذة (آخر N من الأدوار)، والملخص (التاريخ المضغوط)، والمتجهات (الاسترجاع القائم على التضمينات). أنظمة الإنتاج تستخدم ذاكرة هرمية تجمع بين الثلاثة. حدد دائماً ميزانيات توكنات لكل مكون ذاكرة واترك 25% على الأقل من نافذة السياق لرد النموذج.
