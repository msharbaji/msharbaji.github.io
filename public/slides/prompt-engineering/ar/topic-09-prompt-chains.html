قوالب البرومبت والتسلسل

ابنِ مكونات برومبت قابلة لإعادة الاستخدام وسلسلها في سير عمل متعدد الخطوات وقوي.

1

لماذا القوالب؟

مع نمو نظام الذكاء الاصطناعي الخاص بك، ستكتب نفس البرومبتات مراراً وتكراراً بمدخلات مختلفة. قوالب البرومبت تحل هذه المشكلة — فهي قابلة لإعادة الاستخدام ومتسقة وأسهل في الصيانة.

مبدأ DRY

DRY = "Don't Repeat Yourself" (لا تكرر نفسك). كتابة البرومبتات مباشرة في الكود تنتهك هذا المبدأ. إذا قررت تغيير برومبت النظام، يجب أن تجد وتحدّث كل ملف. مع القوالب، تغيّر مكاناً واحداً وينتشر التغيير في كل مكان.

فوائد القوالب

إعادة الاستخدام عبر قاعدة الكود بالكامل. اتساق في برومبتات النظام. سهولة الاختبار وإدارة الإصدارات. تقليل الأخطاء الناتجة عن النسخ واللصق. فصل واضح بين منطق البرومبت ومنطق التطبيق.

2

بناء قوالب البرومبت

هناك عدة طرق لبناء القوالب. لكل منها مزايا وعيوب. دعنا نقارن ثلاث طرق شائعة.

الخيار 1: f-strings في بايثون

بسيطة، بدون تبعيات، لكنها محدودة للمنطق المعقد.

Python — f-string Template 
TEMPLATE = """
You are a data analyst.

Analyze this dataset:

{data}

Focus on: {focus_areas}
"""

result = TEMPLATE.format(
    data="10,20,30,40,50",
    focus_areas="trends and outliers"
)

الخيار 2: Jinja2

أقوى بكثير. تدعم الحلقات والشروط والمرشحات. المعيار الصناعي للقوالب.

Python — Jinja2 Template 
from jinja2 import Template

template_str = """
You are a data analyst.

Analyze this dataset:
{% for item in data %}
- {{ item }}
{% endfor %}

{% if include_summary %}
Include a summary.
{% endif %}
"""

template = Template(template_str)
result = template.render(
    data=[10, 20, 30],
    include_summary=True
)

الخيار 3: LangChain PromptTemplate

مصممة لسير عمل النماذج اللغوية. تتكامل مع سلاسل وأدوات LangChain. مناسبة لأنظمة الإنتاج.

Python — LangChain PromptTemplate 
from langchain.prompts import PromptTemplate

template = PromptTemplate(
    input_variables=["data", "focus"],
    template="""
You are a data analyst.

Analyze this dataset:
{data}

Focus on: {focus}
"""
)

prompt = template.format(
    data="10,20,30,40,50",
    focus="trends"
)

متغيرات القالب والتحقق

عندما تحتوي القوالب على متغيرات، تحتاج إلى التحقق من المدخلات. البيانات المفقودة أو غير الصالحة تسبب برومبتات سيئة.

Python — Validated Template 
from typing import Optional
from dataclasses import dataclass

@dataclass
class AnalysisRequest:
    """Request with validation"""
    dataset: str
    focus_areas: list[str]
    max_length: 500 = 500

    def validate(self):
        if not self.dataset:
            raise ValueError(
                "dataset required"
            )
        if len(self.dataset) > 5000:
            raise ValueError(
                "dataset too large"
            )
        if not self.focus_areas:
            self.focus_areas = ["general trends"]

TEMPLATE = """You are a data analyst.

Analyze:
{dataset}

Focus: {focus}"""

def analyze(request: AnalysisRequest):
    request.validate()
    prompt = TEMPLATE.format(
        dataset=request.dataset,
        focus=", ".join(request.focus_areas)

نمط الإنتاج

استخدم dataclasses أو نماذج Pydantic لكائنات الطلب. تحقق من المدخلات قبل إنشاء البرومبت. ضع قيماً افتراضية معقولة للحقول الاختيارية. هذا يمنع وصول برومبتات مشوهة إلى Claude.

4

أنماط تسلسل البرومبت

سلاسل البرومبت تربط عدة برومبتات في تسلسلات. مخرجات أحدها تصبح مدخلات للتالي. هذا يمكّن من سير عمل معقد يصعب تحقيقه ببرومبت واحد.

→

السلسلة التتابعية

الخطوة 1 → الخطوة 2 → الخطوة 3. كل خطوة تستخدم مخرجات الخطوة السابقة. الأفضل لسير العمل متعدد الخطوات.

⤴️

السلسلة المتوازية

تشغيل عدة برومبتات في نفس الوقت ثم الدمج. ممتازة للتحليل من عدة وجهات نظر.

🔀

السلسلة الشرطية

التوجيه إلى سلاسل مختلفة بناءً على مخرجات النموذج. مثلاً: "هل هذا إيجابي أم سلبي؟" → معالجة مختلفة.

🚦

سلسلة التوجيه

النموذج يختار أي سلسلة يستخدم بناءً على المدخلات. "هذا يتعلق بـ X، لذا استخدم معالج X."

مثال: السلسلة التتابعية

استخراج النقاط الرئيسية → التنظيم حسب الموضوع → توليد الملخص → فحص الجودة. كل خطوة تحسّن مخرجات الخطوة السابقة.

مثال: سلسلة التوجيه

مدخلات المستخدم → الموجّه يحدد الموضوع → إذا "تقني"، استخدم المعالج التقني؛ إذا "عام"، استخدم المعالج العام.

5

بناء سلسلة: ملخص المستندات

لنبنِ سلسلة تتابعية كاملة من 4 خطوات لتلخيص المستندات الطويلة.

1

استخراج النقاط الرئيسية

قراءة المستند وتحديد 5-10 أفكار رئيسية. التركيز على المضمون لا الأسلوب.

2

التنظيم حسب الموضوع

تجميع النقاط الرئيسية في محاور منطقية. مثلاً: "المقدمة"، "الأساليب"، "النتائج".

3

توليد الملخص

كتابة ملخص موجز من النقاط المنظمة. فقرة أو فقرتان، بحد أقصى 200 كلمة.

4

فحص الجودة

التحقق من دقة الملخص وتغطيته للأفكار الرئيسية وعدم وجود معلومات ملفقة. تقييم الجودة من 1-5.

Python — Document Summarization Chain 
def summarize_document(doc_text: str):
    """4-step document summarization chain"""

    # Step 1: Extract key points
    extract_prompt = f"""
Extract 5-10 key points from this document:

{doc_text}

Return only the key points, one per line."""

    response1 = client.messages.create(
        model="claude-opus-4-6",
        max_tokens=500,
        messages=[{"role": "user",
                    "content": extract_prompt}]
    )
    key_points = response1.content[0].text

    # Step 2: Organize by theme
    organize_prompt = f"""
Organize these key points into logical themes:

{key_points}

Output format:
[Theme Name]
- point 1
- point 2
"""

    response2 = client.messages.create(
        model="claude-opus-4-6",
        max_tokens=500,
        messages=[{"role": "user",
                    "content": organize_prompt}]
    )
    organized = response2.content[0].text

    # Step 3: Generate summary
    summary_prompt = f"""
Write a 200-word summary from these organized points:

{organized}

Be concise and accurate. No more than 200 words."""

    response3 = client.messages.create(
        model="claude-opus-4-6",
        max_tokens=300,
        messages=[{"role": "user",
                    "content": summary_prompt}]
    )
    summary = response3.content[0].text

    # Step 4: Quality check
    quality_prompt = f"""
Rate this summary on:
1. Accuracy (no fabrications)
2. Completeness (covers main ideas)
3. Clarity (easy to understand)
4. Conciseness (under 200 words)

Summary:
{summary}

Rate each 1-5 and explain any issues."""

    response4 = client.messages.create(
        model="claude-opus-4-6",
        max_tokens=200,
        messages=[{"role": "user",
                    "content": quality_prompt}]
    )
    quality = response4.content[0].text

    return {
        "summary": summary,
        "quality_check": quality
    }

مكتبات التنسيق

للسلاسل المعقدة، توجد مكتبات متخصصة تتعامل مع التعقيد. إليك نظرة سريعة.

🔗

LangChain

الأكثر شعبية. سلاسل وذاكرة ووكلاء وتكامل RAG. توثيق جيد. أثقل لكن غنية بالميزات.

🦙

LlamaIndex

تركز على فهرسة البيانات وRAG. ممتازة لمعالجة المستندات. أخف وزناً من LangChain.

🧠

Haystack

تركز على البحث وRAG. بنية قائمة على المكونات. جيدة لخطوط أنابيب الإنتاج.

🏗️

ابنِ بنفسك

السلاسل البسيطة لا تحتاج إطار عمل. الكود المخصص يمنحك تحكماً كاملاً ويقلل التبعيات.

✅

متى تستخدم كل واحدة

LangChain: وكلاء معقدون متعددو الأدوات، إدارة الذاكرة، تكاملات متعددة. LlamaIndex: أنظمة RAG ثقيلة المستندات. مخصص: سلاسل بسيطة من 2-3 خطوات، أقصى تحكم وأقل تبعيات. ابدأ بالبسيط، وأضف إطار عمل عند الحاجة.

✓

اختبر فهمك

اختبار سريع — 4 أسئلة

1. ما الفائدة الرئيسية من استخدام قوالب البرومبت بدلاً من كتابة البرومبتات مباشرة في الكود؟

2. أي طريقة قوالب هي الأقوى وتدعم الحلقات والشروط؟

3. في سلسلة البرومبت، ماذا يحدث في كل خطوة؟

4. متى يجب استخدام مكتبات التنسيق مثل LangChain؟

✓

ملخص الموضوع 9

قوالب البرومبت تتبع مبدأ DRY — اكتب مرة واحدة واستخدم في كل مكان. يمكنك بناؤها باستخدام f-strings (بسيطة)، أو Jinja2 (قوية)، أو LangChain (إطار عمل)، أو كود مخصص. تحقق دائماً من متغيرات القالب قبل استخدامها. سلاسل البرومبت تربط عدة برومبتات في تسلسلات: تتابعية (خطوة بخطوة)، أو متوازية (عدة وجهات نظر)، أو شرطية (تفرّع)، أو توجيهية (النموذج يختار). السلاسل تمكّن سير عمل معقد يستحيل بأحد البرومبتات الواحدة. للسلاسل البسيطة، اكتب كوداً مخصصاً. للأنظمة المعقدة متعددة الأدوات، استخدم LangChain أو مكتبات مشابهة.
