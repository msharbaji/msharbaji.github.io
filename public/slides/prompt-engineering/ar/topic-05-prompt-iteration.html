تكرار وتصحيح البرومبتات

تعلّم العملية المنهجية لتحسين البرومبتات — من المسودة الأولى إلى الجاهزية للإنتاج.

1

حلقة التكرار

هندسة البرومبت ليست عملية من خطوة واحدة. البرومبتات الممتازة تُبنى من خلال التكرار: اكتب ← اختبر ← حلّل ← حسّن ← كرّر. هذه الدورة تُحسّن جودة المخرجات بشكل أُسّي.

حلقة التكرار (بالتفصيل):

  1. اكتب: صُغ مسودة البرومبت الأولى بناءً على المتطلبات
  2. اختبر: شغّل البرومبت على حالات اختبار تمثيلية (10-20 مثالاً كحد أدنى)
  3. حلّل: افحص حالات الفشل والأنماط. اسأل "لماذا فشل؟"
  4. حسّن: أجرِ تعديلات مستهدفة لمعالجة الأسباب الجذرية
  5. كرّر: استمر حتى تصبح النتائج مقبولة
💡

تشبيه: تطوير البرمجيات

هندسة البرومبت تشبه كتابة الكود. مسودتك الأولى لن تكون مثالية. تختبر وتصحح وتعيد هيكلة وتكرر. الفرق: مع البرومبتات، "اختباراتك" هي استدعاءات API صغيرة، وليست مجموعات اختبار كاملة.

الحلقة عملياً:

Python — Iteration Loop 
import anthropic

client = anthropic.Anthropic()

# Test cases
test_cases = [
    ("I love this!", "positive"),
    ("Terrible product.", "negative"),
    ("It's okay.", "neutral"),
]

def test_prompt(prompt, test_data):
    """Run prompt on test cases and check accuracy."""
    correct = 0
    for text, expected in test_data:
        response = client.messages.create(
            model="claude-sonnet-4-5-20250929",
            max_tokens=100,
            messages=[{
                "role": "user",
                "content": f"{prompt}\n\nText: {text}"
            }]
        )
        result = response.content[0].text.strip().lower()
        if expected in result:
            correct += 1
    accuracy = (correct / len(test_data)) * 100
    return accuracy

# Iteration cycle
prompt_v1 = "Classify sentiment: positive, negative, or neutral."
acc_v1 = test_prompt(prompt_v1, test_cases)
print(f"V1: {acc_v1}%")

# Analyze and improve
prompt_v2 = """Classify sentiment.
Examples: "love" → positive, "bad" → negative.
Return ONE word: positive, negative, or neutral."""
acc_v2 = test_prompt(prompt_v2, test_cases)
print(f"V2: {acc_v2}%")

قِس كل شيء

حدّد مقاييس النجاح مسبقاً (الدقة، زمن الاستجابة، التكلفة). بدون مقاييس، أنت تكرر بشكل أعمى. قِس قبل وبعد كل تعديل.

2

أنماط الفشل الشائعة

معظم حالات فشل البرومبتات تقع في أنماط يمكن التنبؤ بها. تعلّم التعرف عليها، وستصحح الأخطاء بشكل أسرع.

نمط الفشل 1: البرومبتات المُسهبة جداً

❌ المشكلة
Your job is to carefully read the
text and think deeply about whether
the sentiment is positive or negative.
Please consider all nuances...
✅ الحل
Classify sentiment: positive or
negative. Be concise. One word
answer only.

لماذا يفشل: البرومبتات الطويلة تُربك النموذج وتهدر التوكنات. كيف تُصلح: احذف الكلمات الزائدة. كن محدداً، لا مُسهباً.

نمط الفشل 2: الهلوسات

❌ المشكلة
What is the annual revenue of
company X?
✅ الحل
Based ONLY on this text, extract
the annual revenue. If not mentioned,
say "Not found".

لماذا يفشل: النموذج يختلق معلومات عندما يكون غير متأكد. كيف تُصلح: أخبره بما يفعل عندما لا تتوفر المعلومات.

نمط الفشل 3: تنسيق المخرجات الخاطئ

❌ المشكلة
Return the data as JSON.
✅ الحل
Return valid JSON (no markdown).
{
  "name": "...",
  "score": 0-100
}

لماذا يفشل: كلمة "JSON" غامضة. قد يُرجع النموذج JSON ملفوفاً بـ markdown. كيف تُصلح: أظهر التنسيق المتوقع بدقة مع مثال.

نمط الفشل 4: تجاهل التعليمات

❌ المشكلة
Don't be too creative. Just summarize
the text factually.
✅ الحل
Summarize ONLY facts from the text.
Do NOT add interpretations or
creative additions.

لماذا يفشل: النفي الضعيف ("don't") أقل فعالية. كيف تُصلح: استخدم برومبت النظام للقواعد الصارمة. كن صريحاً.

نمط الفشل 5: الردود الكسولة

❌ المشكلة
Summarize the meeting.
✅ الحل
Summarize the meeting covering:
1. Key decisions (at least 3)
2. Action items with owners
3. Next meeting date
Format: markdown with headers

لماذا يفشل: التعليمات الغامضة تسمح بردود منخفضة الجهد. كيف تُصلح: حدّد بدقة ما تريده. ضع توقعات واضحة.

⚠️

اختبر مقابل أنماط الفشل

عندما تلاحظ فشلاً، حدّد أي نمط فشل هو. ثم طبّق الحل المستهدف. لا تقل فقط "البرومبت سيئ" — شخّص المشكلة المحددة.

3

تقنيات التصحيح

عندما يفشل برومبت، تحتاج لعزل السبب. هل هو برومبت النظام؟ بيانات الإدخال؟ مواصفات تنسيق المخرجات؟ التصحيح هو عمل تحقيقي.

عملية التصحيح المنهجي:

  1. اعزل: اختبر كل مكوّن على حدة (برومبت النظام، الأمثلة، مواصفات التنسيق)
  2. بسّط: أنشئ حالات اختبار بسيطة تعيد إنتاج الفشل
  3. افترض: كوّن نظرية حول ما هو خطأ
  4. اختبر: أجرِ تغييراً واحداً وقِس التأثير
  5. كرّر: إذا كان الافتراض صحيحاً، طبّقه على جميع الحالات

مثال تصحيح: "النموذج يُرجع markdown عندما أريد JSON"

تغيير واحد في المرة

لا تغيّر أشياء متعددة في وقت واحد. غيّر متغيراً واحداً، قِس، ثم انتقل للتالي. وإلا لن تعرف أي تغيير سبّب التحسن.

4

اختبار A/B للبرومبتات

عندما يكون لديك نسختان متنافستان من البرومبت، اختبرهما A/B على مجموعة بياناتك. لا تعتمد على الحدس. دع البيانات تقرر.

عملية اختبار A/B:

  1. أنشئ مجموعة بيانات اختبار (50-200 مثال بإجابات صحيحة معروفة)
  2. شغّل البرومبت A على جميع الأمثلة، وقِس الدقة/الجودة
  3. شغّل البرومبت B على جميع الأمثلة، وقِس نفس المقاييس
  4. قارن: أيهما أداؤه أفضل؟ بكم؟
  5. إذا كان B أفضل، اعتمده. إذا كان متعادلاً أو بفارق هامشي، ابقَ مع A (الأبسط).

Python: كود اختبار A/B

Python — A/B Testing Framework 
import json
from anthropic import Anthropic

client = Anthropic()

# Test dataset
test_data = [
    {"text": "I love this!", "expected": "positive"},
    {"text": "Awful experience.", "expected": "negative"},
    {"text": "It's okay.", "expected": "neutral"},
]

def test_prompt_version(prompt, dataset):
    """Test a prompt and return accuracy."""
    results = []
    for item in dataset:
        response = client.messages.create(
            model="claude-sonnet-4-5-20250929",
            max_tokens=100,
            messages=[{
                "role": "user",
                "content": f"{prompt}\n\nText: {item['text']}"
            }]
        )
        result = response.content[0].text.strip().lower()
        correct = item["expected"] in result
        results.append({
            "input": item["text"],
            "expected": item["expected"],
            "output": result,
            "correct": correct
        })
    accuracy = (sum(1 for r in results if r["correct"])) / len(results) * 100
    return accuracy, results

# Prompt A (basic)
prompt_a = "Classify sentiment: positive, negative, or neutral."

# Prompt B (improved with examples)
prompt_b = """Classify sentiment: positive, negative, neutral.
Examples:
- "love" → positive
- "hate" → negative
- "ok" → neutral

Return ONE word only."""

# Run A/B test
acc_a, results_a = test_prompt_version(prompt_a, test_data)
acc_b, results_b = test_prompt_version(prompt_b, test_data)

print(f"Prompt A accuracy: {acc_a}%")
print(f"Prompt B accuracy: {acc_b}%")
print(f"Winner: {'B' if acc_b > acc_a else 'A'}")
✅

الأهمية الإحصائية

التحسنات الصغيرة (1-2%) قد تكون ضوضاء. ابحث عن فجوات أكبر (5%+) قبل إعلان فائز. على مجموعات بيانات أكبر (1000+ مثال)، التحسنات الصغيرة تصبح ذات معنى.

5

إدارة إصدارات البرومبتات

تتبّع إصدارات برومبتاتك كما تتتبع الكود. إدارة الإصدارات تتيح لك المقارنة، والتراجع، وفهم ما تغيّر ولماذا. هذا أمر أساسي في الإنتاج.

مثال على سجل تغييرات البرومبت:

Text — Prompt Changelog 
## Sentiment Classifier Prompt Changelog

### v1.0 (2025-01-15)
- Initial: "Classify sentiment: positive, negative, neutral"
- Accuracy: 72%
- Issue: Too vague, model guesses

### v1.1 (2025-01-16)
- Added few-shot examples
- Added "Return ONE word only" constraint
- Accuracy: 85%
- Improvement: +13%

### v1.2 (2025-01-17)
- Moved examples to system prompt
- Added "If unsure, say 'unclear'"
- Accuracy: 87%
- Improvement: +2%

### v2.0 (2025-01-18)
- Full XML structure with tags
- Explicit JSON output format
- Accuracy: 89%
- Improvement: +2%, better production-grade

أفضل الممارسات لإدارة الإصدارات:

  • الإصدار الدلالي: v1.0, v1.1, v2.0 (رئيسي.فرعي.تصحيحي)
  • الطوابع الزمنية: أضف التاريخ لتتبع متى حدثت التغييرات
  • المقاييس: سجّل دائماً مقاييس الدقة/الجودة
  • المبرر: اشرح لماذا أجريت كل تغيير
  • المقارنة: أظهر المقاييس قبل/بعد لإثبات التحسن
💡

خزّن البرومبتات في نظام التحكم بالإصدارات

احفظ البرومبتات في Git (أو ما يعادله) بجانب كودك. ضع علامات للإصدارات. هذا يتيح لك التراجع لإصدار سابق إذا كان برومبت جديد أقل أداءً.

دراسة حالة واقعية: برومبت مراجعة الكود

لنستعرض مثالاً حقيقياً لتكرار برومبت مراجعة كود من سيئ (v1) إلى ممتاز (v5). لاحظ نمط التحسينات التدريجية.

Prompt — V1: Too Vague

التدرج

لاحظ أن كل إصدار يبني على السابق: v1 ← v2 (إضافة سياق) ← v3 (هيكلة) ← v4 (علامات XML، حواجز حماية) ← v5 (أمثلة). هذا هو المسار الذي تسلكه معظم البرومبتات نحو الإنتاج.

اختبر فهمك

اختبار سريع — 4 أسئلة

1. ما هي خطوات حلقة تكرار البرومبت؟

2. أي نمط فشل يُعالَج بإضافة "Return ONLY JSON. No markdown"؟

3. عند تصحيح برومبت، ماذا يجب أن تفعل؟

4. لماذا يجب أن تدير إصدارات برومبتاتك؟

ملخص الموضوع 5

إليك ما تعلمته:

التكرار هو كل شيء. ابدأ ببساطة، اختبر، وحسّن. أنماط الفشل يمكن التنبؤ بها. تعلّم التعرف عليها وإصلاحها. التصحيح منهجي. اعزل، افترض، اختبر تغييراً واحداً في كل مرة. اختبار A/B يلغي التخمين. دع البيانات تقرر بين الإصدارات. إدارة الإصدارات تمنع الفوضى. تتبع التغييرات مثل الكود. دراسات الحالة تُظهر المسار. معظم البرومبتات تتبع تدرجاً من v1 إلى v5 نحو الإنتاج.
