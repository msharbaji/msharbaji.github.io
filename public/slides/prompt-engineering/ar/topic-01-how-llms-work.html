ما هو النموذج اللغوي الكبير؟

النموذج اللغوي الكبير (LLM) هو شبكة عصبية تم تدريبها على كميات هائلة من البيانات النصية. مهمته الأساسية بسيطة بشكل مخادع: بالنظر إلى تسلسل من النص، يتنبأ بالكلمة (التوكن) الأكثر احتمالاً التالية.

هذا كل شيء. كل ما رأيته من إنجازات مبهرة — كتابة الكود، والإجابة على الأسئلة، والاستدلال الرياضي — ينبع من هذه الآلية الوحيدة، بعد توسيعها بشكل هائل.

💡

تشبيه: أفضل إكمال تلقائي في العالم

تخيل خاصية الإكمال التلقائي في هاتفك، لكنها مدرّبة على تريليونات الكلمات من الكتب والأكواد والمواقع والمحادثات. لا تتنبأ فقط بالكلمة التالية — بل تعلمت أنماط المنطق والقواعد والحقائق والاستدلال من كل تلك البيانات.

التوكنات: لغة النماذج اللغوية الكبيرة

النماذج اللغوية الكبيرة لا تقرأ الكلمات كما يفعل البشر. بل تقسّم النص إلى توكنات — أجزاء قد تكون كلمة، أو جزءاً من كلمة، أو حتى حرفاً واحداً. تسمى هذه العملية التوكنة (Tokenization).

🔡

الكلمات الشائعة = توكن واحد

"hello" → [hello]
"the" → [the]
الكلمات البسيطة تبقى كاملة.

✂️

الكلمات الطويلة = عدة توكنات

"extraordinary" → [extra][ordinary]
"tokenization" → [token][ization]

🔢

الأرقام تُقسَّم

"12345" → [123][45]
لهذا السبب تعاني النماذج مع الحسابات الدقيقة!

📏

~1 توكن ≈ 4 أحرف

كقاعدة تقريبية: التوكن الواحد يساوي حوالي 4 أحرف أو ¾ كلمة باللغة الإنجليزية.

لماذا التوكنات مهمة لهندسة البرومبت

كل نموذج لغوي كبير لديه حد توكنات (نافذة السياق). Gemini 2.5 Pro لديه مليون توكن. Claude Sonnet/Opus لديه ~200 ألف توكن. GPT-4o لديه ~128 ألف توكن. يجب أن يتسع البرومبت + رد النموذج ضمن هذا الحد. برومبتات أطول = تكلفة أعلى ووقت استجابة أطول. كفاءة التوكنات مهارة أساسية.

نافذة السياق

نافذة السياق هي "الذاكرة العاملة" للنموذج — إجمالي كمية النص (بالتوكنات) التي يمكنه رؤيتها دفعة واحدة. تشمل برومبت النظام، وتاريخ المحادثة، والرد الذي يتم توليده.

فكرة جوهرية

النموذج لا يملك ذاكرة بين المحادثات. كل استدعاء API يبدأ من الصفر. كل ما "يعرفه" النموذج عن محادثتك يجب أن يكون داخل نافذة السياق. لهذا السبب ترسل تطبيقات المحادثة تاريخ المحادثة بالكامل مع كل رسالة.

4

الحرارة والعينات: التحكم بالإبداع

عندما يتنبأ النموذج بالتوكن التالي، لا يختار ببساطة الأفضل. بل يحسب توزيع احتمالي على كل التوكنات التالية الممكنة. ثم يأخذ عينة من هذا التوزيع. معامل الحرارة (temperature) يتحكم بكيفية ذلك.

المعاملالقيمةالتأثير
temperature 0.0 يختار دائماً التوكن الأكثر احتمالاً. حتمي ومركّز ومتكرر.
temperature 0.7 متوازن — إبداعي لكن متسق. القيمة الافتراضية الجيدة لمعظم المهام.
temperature 1.0+ عشوائي جداً. مناسب للكتابة الإبداعية والعصف الذهني. قد يكون غير متسق.
top_p 0.0–1.0 يأخذ بالاعتبار فقط التوكنات في أعلى P% من كتلة الاحتمالات. بديل للحرارة.
max_tokens عدد صحيح الحد الأقصى لعدد التوكنات التي سيولدها النموذج في رده.

أول استدعاء API (بايثون)

بما أنك تعرف بايثون، دعنا نرى كيف يبدو استدعاء API فعلي لنموذج لغوي. إليك كيفية استدعاء Claude باستخدام Anthropic SDK:

Python 
# Install: pip install anthropic
import anthropic

# Create client (uses ANTHROPIC_API_KEY env variable)
client = anthropic.Anthropic()

# Make an API call
message = client.messages.create(
    model="claude-sonnet-4-5-20250929",
    max_tokens=1024,
    temperature=0.7,
    system="You are a helpful coding tutor.",
    messages=[
        {"role": "user", "content": "Explain Python decorators in 3 sentences."}
    ]
)

print(message.content[0].text)
# Also useful:
print(f"Tokens used: {message.usage.input_tokens} in, {message.usage.output_tokens} out")

وهذا هو المكافئ مع OpenAI API للمقارنة:

Python 
# Install: pip install openai
from openai import OpenAI

client = OpenAI()  # uses OPENAI_API_KEY

response = client.chat.completions.create(
    model="gpt-4o",
    temperature=0.7,
    max_tokens=1024,
    messages=[
        {"role": "system", "content": "You are a helpful coding tutor."},
        {"role": "user", "content": "Explain Python decorators in 3 sentences."}
    ]
)

print(response.choices[0].message.content)
✅

لاحظ النمط

كلا الـ APIs يتشاركان نفس البنية الأساسية: model (أي نموذج لغوي)، messages (المحادثة)، وparameters (الحرارة، الحد الأقصى للتوكنات). هذا النمط عالمي عبر جميع مزودي النماذج اللغوية.

6

النماذج الذهنية الأساسية

قبل أن ننتقل لتقنيات هندسة البرومبت، استوعب هذه النماذج الذهنية. ستشكّل طريقة تفكيرك في كل برومبت تكتبه.

🎰

إنه احتمالي

النماذج اللغوية عشوائية — نفس البرومبت يمكن أن ينتج مخرجات مختلفة. هذا ليس خطأ، بل ميزة يمكنك التحكم بها عبر الحرارة.

🪞

مدخلات سيئة = مخرجات سيئة

جودة المخرجات تتناسب طردياً مع جودة المدخلات. برومبتات غامضة → إجابات غامضة.

📦

لا حالة مخفية

النموذج يعرف فقط ما هو موجود في نافذة السياق. ليس لديه ذاكرة ولا ملفات ولا وصول للإنترنت (إلا إذا أعطيته أدوات).

🎭

إنه لاعب أدوار

النماذج اللغوية مدرّبة على إكمال النصوص. إذا أعددت سيناريو (برومبت نظام)، سيلعب ذلك الدور. هذا أساس هندسة البرومبت.

✓

اختبر فهمك

اختبار سريع — 3 أسئلة

1. ما الذي يفعله النموذج اللغوي الكبير بشكل أساسي؟

2. إذا أردت مخرجات حتمية ومركّزة (مثل تحليل JSON)، أي حرارة يجب أن تستخدم؟

3. لماذا تعاني النماذج اللغوية أحياناً مع الحسابات مثل "ما حاصل 7,391 × 4,528؟"

✓

ملخص الموضوع 1

إليك ما تعلمته:

النماذج اللغوية الكبيرة هي متنبئات بالتوكن التالي مدرّبة على بيانات نصية ضخمة. تعالج النص على شكل توكنات (~4 أحرف لكل توكن) ضمن نافذة سياق محددة. تتحكم بعشوائية المخرجات عبر الحرارة (0 = حتمي، 1+ = إبداعي). النموذج لا يملك ذاكرة بين الاستدعاءات — كل شيء يجب أن يكون في نافذة السياق.
