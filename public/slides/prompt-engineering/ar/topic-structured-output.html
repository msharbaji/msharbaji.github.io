المخرجات المنظمة وتحليلها

احصل على JSON موثوق، وجداول، وبيانات منظمة من النماذج اللغوية — باستخدام المخططات، والتحقق، واستراتيجيات التحليل.

1

لماذا المخرجات المنظمة مهمة

النماذج اللغوية الكبيرة تنتج بطبيعتها نصوصاً حرة — كلاماً متدفقاً وحوارياً. لكن التطبيقات الحقيقية تحتاج بيانات منظمة: JSON لواجهات البرمجة، وصفوف لقواعد البيانات، وكائنات محددة الأنواع لمنطق التطبيق. الفجوة بين النص الحر والبيانات القابلة للقراءة آلياً هي المجال الذي تتدخل فيه تقنيات المخرجات المنظمة.

بدون المخرجات المنظمة، ستضطر لكتابة كود هش لتحليل النصوص يتعطل كلما غيّر النموذج صياغته. مع المقاربة الصحيحة، تحصل على بيانات متوقعة ومُتحقق منها في كل مرة.

📦

استجابات JSON

إرجاع البيانات ككائنات JSON صالحة يمكن لتطبيقك تحليلها واستخدامها مباشرة دون معالجة نصية.

🔍

استخراج البيانات

سحب حقول منظمة (أسماء، تواريخ، مبالغ) من نصوص غير منظمة كالرسائل الإلكترونية والمستندات والتقارير.

🔗

التكامل مع واجهات البرمجة

توليد مخرجات تطابق مخططات واجهات البرمجة الخاصة بك حتى تتدفق استجابات النموذج مباشرة إلى نظامك.

🗄️

تعبئة قواعد البيانات

تحويل اللغة الطبيعية إلى سجلات منظمة جاهزة للإدراج في قاعدة البيانات مع أنواع وقيود صحيحة.

💡

تشبيه: نموذج مقابل صفحة فارغة

فكر في المخرجات المنظمة كإعطاء النموذج اللغوي نموذجاً (فورم) لملئه بدلاً من صفحة فارغة. الصفحة الفارغة تدعو للإسهاب. أما النموذج ذو الحقول المسمّاة والقوائم المنسدلة وخانات الاختيار فيحصل لك على البيانات التي تحتاجها بالضبط، بالتنسيق الذي تتوقعه.

2

وضع JSON وتنسيق الاستجابة

مزودو النماذج اللغوية المختلفون يوفرون آليات مختلفة لطلب المخرجات المنظمة. بعضها لديه أوضاع JSON أصلية مدمجة في واجهات البرمجة، بينما يعتمد البعض الآخر على صياغة دقيقة للبرومبت. إليك كيف يتعامل المزودون الرئيسيون مع الأمر:

OpenAI — وضع JSON أصلي عبر response_format:

Python 
from openai import OpenAI

client = OpenAI()

response = client.chat.completions.create(
    model="gpt-4o",
    response_format={"type": "json_object"},
    messages=[
        {"role": "system", "content": "Return JSON with keys: title, summary, rating"},
        {"role": "user", "content": "Review the movie Inception"}
    ]
)

# response.choices[0].message.content is guaranteed valid JSON
import json
data = json.loads(response.choices[0].message.content)
print(data["title"])  # "Inception"

Anthropic — مخرجات منظمة عبر برومبت النظام + وسوم XML أو استخدام الأدوات:

Python 
import anthropic
import json

client = anthropic.Anthropic()

message = client.messages.create(
    model="claude-sonnet-4-5-20250929",
    max_tokens=1024,
    system="""Return your response as a JSON object inside <json> tags.
The JSON must have these exact keys: title, summary, rating.
Example: <json>{"title": "...", "summary": "...", "rating": 8.5}</json>""",
    messages=[
        {"role": "user", "content": "Review the movie Inception"}
    ]
)

# Extract JSON from XML tags
import re
match = re.search(r'<json>([\s\S]*?)</json>', message.content[0].text)
data = json.loads(match.group(1))

Google Gemini — توليد مبني على المخطط مع نوع MIME للاستجابة:

Python 
import google.generativeai as genai

model = genai.GenerativeModel("gemini-2.5-flash")

response = model.generate_content(
    "Review the movie Inception",
    generation_config=genai.types.GenerationConfig(
        response_mime_type="application/json",
        response_schema={
            "type": "object",
            "properties": {
                "title": {"type": "string"},
                "summary": {"type": "string"},
                "rating": {"type": "number"}
            },
            "required": ["title", "summary", "rating"]
        }
    )
)
✅

فكرة جوهرية

ليست كل النماذج لديها وضع JSON أصلي — لكن يمكنك الحصول على JSON موثوق من أي نموذج بالصياغة الصحيحة. المفتاح هو أن تكون صريحاً بشأن البنية المتوقعة، وأن تقدم مثالاً، وأن تستخدم محددات (مثل وسوم XML) حتى تتمكن من استخراج الـ JSON برمجياً.

3

التوليد المبني على المخططات

أكثر المقاربات موثوقية للمخرجات المنظمة هي التوليد المبني على المخططات — تعريف الشكل الدقيق لبياناتك بالأنواع والقيود والأوصاف، ثم ترك مكتبة تفرض ذلك المخطط على مخرجات النموذج. هنا تتألق أدوات مثل Pydantic وInstructor.

أولاً، عرّف نموذج البيانات باستخدام Pydantic:

Python 
from pydantic import BaseModel, Field

class MovieReview(BaseModel):
    title: str = Field(description="Movie title")
    rating: float = Field(ge=1, le=10, description="Rating from 1-10")
    sentiment: str = Field(description="positive, negative, or neutral")
    key_themes: list[str] = Field(description="Main themes")
    summary: str = Field(max_length=200)

ثم استخدم مكتبة Instructor للحصول على استجابات محددة الأنواع ومُتحقق منها تلقائياً:

Python 
# Install: pip install instructor openai
import instructor
from openai import OpenAI

# Patch the OpenAI client with Instructor
client = instructor.from_openai(OpenAI())

# Get a typed, validated response — no manual parsing!
review = client.chat.completions.create(
    model="gpt-4o",
    response_model=MovieReview,
    messages=[
        {"role": "user", "content": "Review the movie Inception"}
    ]
)

# review is a fully typed MovieReview object
print(review.title)       # "Inception"
print(review.rating)      # 9.2
print(review.sentiment)   # "positive"
print(review.key_themes)  # ["dreams", "reality", "loss"]

# Validation is automatic — rating must be 1-10, summary ≤ 200 chars
# If the LLM returns invalid data, Instructor retries automatically
✅

لماذا المقاربة المبنية على المخططات هي الأفضل

نموذج Pydantic يخدم ثلاثة أغراض في آن واحد: يوثّق التنسيق المتوقع، ويقيّد مخرجات النموذج، ويتحقق من النتيجة. إذا أرجع النموذج تقييماً قدره 15، يكتشف Pydantic المخالفة ويعيد Instructor المحاولة تلقائياً مع تغذية راجعة تصحيحية.

4

استراتيجيات التحليل

في بيئة الإنتاج، استجابات النماذج اللغوية ليست دائماً منسقة بشكل مثالي. قد يغلف النموذج JSON في كتل كود markdown، أو يضيف نصاً توضيحياً قبل JSON، أو ينتج مخرجات مشوهة قليلاً. استراتيجية التحليل القوية تتعامل مع كل هذه الحالات بسلاسة.

إليك دالة تحليل مجرّبة ميدانياً تتعامل مع أكثر الحالات الحدية شيوعاً:

Python 
import json
import re

def extract_json(text: str) -> dict:
    """Extract JSON from LLM response, handling markdown code blocks."""
    # Try direct parse first
    try:
        return json.loads(text)
    except json.JSONDecodeError:
        pass

    # Try extracting from markdown code block
    match = re.search(r'```(?:json)?\s*([\s\S]*?)```', text)
    if match:
        return json.loads(match.group(1).strip())

    # Try finding JSON-like substring
    match = re.search(r'\{[\s\S]*\}', text)
    if match:
        return json.loads(match.group(0))

    raise ValueError("No valid JSON found in response")

لمزيد من المتانة، أضف حلقة إعادة المحاولة مع التغذية الراجعة التي ترسل أخطاء التحليل مرة أخرى للنموذج:

Python 
def get_json_with_retry(client, messages, max_retries=3):
    """Get valid JSON from LLM with automatic retry on parse failure."""
    for attempt in range(max_retries):
        response = client.messages.create(
            model="claude-sonnet-4-5-20250929",
            max_tokens=1024,
            messages=messages
        )
        text = response.content[0].text

        try:
            return extract_json(text)
        except (json.JSONDecodeError, ValueError) as e:
            # Send the error back to the model for correction
            messages.append({"role": "assistant", "content": text})
            messages.append({
                "role": "user",
                "content": f"That response failed to parse: {e}\n"
                           f"Please return ONLY valid JSON, no other text."
            })

    raise ValueError(f"Failed to get valid JSON after {max_retries} attempts")
⚠️

تحقق دائماً من المخطط

تحقق دائماً من المخرجات المُحللة مقابل المخطط المتوقع — النماذج اللغوية قد تنتج JSON صالحاً لا يطابق بنيتك. استجابة مثل {"result": "success"} هي JSON صالح لكنه عديم الفائدة إذا كنت تتوقع {"title": "...", "rating": 9}. استخدم Pydantic أو JSON Schema للتحقق من التطابق البنيوي، وليس فقط أخطاء الصياغة.

5

بث المخرجات المنظمة

بث استجابات النماذج اللغوية رائع لتجربة المستخدم — يرى المستخدمون النص يظهر في الوقت الحقيقي. لكن بث البيانات المنظمة يطرح تحدياً: لا يمكنك تحليل JSON غير مكتمل. استجابة جزئية مثل {"title": "Incep ستسبب خطأ في التحليل.

هناك استراتيجيتان رئيسيتان لبث المخرجات المنظمة:

📝

JSON مفصول بالأسطر (NDJSON)

كل سطر هو كائن JSON كامل. حلّل سطراً بسطر فور وصولها. مثالي لقوائم العناصر أو المخرجات بنمط السجلات.

🧩

تحليل الكائنات الجزئية

استخدم مكتبات يمكنها استخراج الحقول المكتملة من JSON غير مكتمل، مع تحديث واجهة المستخدم كلما أصبح حقل متاحاً.

Python 
import instructor
from openai import OpenAI
from pydantic import BaseModel

class MovieReview(BaseModel):
    title: str
    rating: float
    summary: str

client = instructor.from_openai(OpenAI())

# Stream partial objects — fields populate as they're generated
review_stream = client.chat.completions.create_partial(
    model="gpt-4o",
    response_model=MovieReview,
    messages=[
        {"role": "user", "content": "Review the movie Inception"}
    ]
)

for partial_review in review_stream:
    # partial_review has fields filled in as they stream
    print(f"Title: {partial_review.title or '...'}")
    print(f"Rating: {partial_review.rating or '...'}")
    print(f"Summary: {partial_review.summary or '...'}")
    print("---")
✅

متى يكون البث مهماً

بث المخرجات المنظمة أكثر قيمة لواجهات المستخدم الحية حيث يشاهد المستخدمون النتائج تظهر (لوحات المعلومات، نتائج البحث، جداول البيانات) وللمخرجات الكبيرة حيث الانتظار للاستجابة الكاملة سيسبب تأخيراً غير مقبول. لخطوط الأنابيب الخلفية، التحليل الدفعي بعد الاكتمال أبسط ومفضّل.

أفضل الممارسات

اتبع هذه المبادئ لبناء خطوط أنابيب مخرجات منظمة موثوقة تعمل في بيئة الإنتاج.

📐

عرّف مخططاً دائماً

المخططات الصريحة تعطي نتائج أفضل من الأمل في التنسيق الصحيح. استخدم نماذج Pydantic أو JSON Schema أو على الأقل مثالاً واضحاً في برومبتك.

🛡️

تحقق ولا تثق

تحقق دائماً من المخرجات مقابل مخططك قبل استخدامها. JSON صالح لا يعني بنية صحيحة. تحقق من الأنواع والحقول المطلوبة وقيود القيم.

🔄

أعد المحاولة مع التغذية الراجعة

إذا فشل التحليل، أرسل الخطأ مرة أخرى للنموذج للتصحيح. معظم النماذج تصحح نفسها بشكل موثوق عند إظهار خطئها والتنسيق المتوقع.

🎯

ابدأ ببساطة

ابدأ ببنى مسطحة؛ أضف التعشيش فقط عند الحاجة. المخططات المتداخلة بعمق أصعب على النماذج وأصعب عليك للتحقق منها.

اختبر فهمك

اختبار سريع — 3 أسئلة

1. ما أكثر طريقة موثوقة للحصول على مخرجات JSON من نموذج لغوي؟

2. لماذا يجب التحقق من مخرجات النموذج حتى عند استخدام وضع JSON؟

3. ماذا يجب أن تفعل عند فشل تحليل JSON من استجابة نموذج لغوي؟
