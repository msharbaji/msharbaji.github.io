RAG: التوليد المعزز بالاسترجاع

امنح نموذجك اللغوي وصولاً للمعرفة الخارجية — أهم نمط في أنظمة الذكاء الاصطناعي الإنتاجية.

1

المشكلة التي يحلها RAG

تعاني النماذج اللغوية الكبيرة من ثلاث قيود جوهرية يعالجها RAG:

🕐

معرفة قديمة

بيانات تدريب النموذج لها تاريخ انتهاء. معرفة Claude Opus تنتهي في أوائل 2025. لا يستطيع الإجابة عن أحداث وقعت بعد ذلك.

🤥

الهلوسات

عندما لا تعرف النماذج شيئاً، تختلق إجابات تبدو معقولة بثقة تامة. أعطها حقائق حقيقية، وتنخفض الهلوسات بشكل كبير.

🔒

لا وصول للبيانات الخاصة

لا تستطيع النماذج قراءة مستندات شركتك الداخلية أو قاعدة الكود أو بيانات العملاء. تعمل فقط مع ما تعطيها إياه في البرومبت.

💰

كفاءة التوكنات

لا يمكنك وضع قاعدة معرفتك بالكامل في برومبت النظام. يتيح لك RAG استرجاع المستندات ذات الصلة فقط ديناميكياً، مما يوفر التوكنات والتكلفة.

✅

RAG = استرجاع + توليد بالنموذج اللغوي

بدلاً من الأمل بأن النموذج يعرف شيئاً ما، تسترجع المستندات ذات الصلة، وتحقنها في البرومبت، ثم تطلب من النموذج الإجابة بناءً على تلك الحقائق. هذا هو أهم نمط في أنظمة الذكاء الاصطناعي الإنتاجية.

الخطوة 1: الاستعلام (Query)

يطرح المستخدم سؤالاً. يُمرر هذا الاستعلام إلى نظام الاسترجاع. لا يُرسل الاستعلام نفسه إلى Claude بعد — بل يُستخدم للعثور على المستندات ذات الصلة.

الخطوة 2: الاسترجاع (Retrieve)

يجد نظام الاسترجاع (عادةً قاعدة بيانات متجهات مثل Pinecone أو Weaviate أو Chroma) المستندات المشابهة للاستعلام. الأكثر شيوعاً هو استخدام البحث الدلالي — تحويل الاستعلام إلى أرقام (embeddings) والعثور على المستندات القريبة.

الخطوة 3: التعزيز (Augment)

تُحقن المستندات المسترجعة في البرومبت. شكل التعزيز النموذجي يبدو هكذا:

Augmented Prompt 
You are a helpful customer support agent.

<context>
The following documents are relevant to the customer's question:

---
From our Knowledge Base (Product FAQ):
Q: How do I reset my password?
A: Click "Forgot Password" on the login page, enter your email,
   and follow the verification link.
---

From our Support Docs:
Resetting passwords typically takes 5-10 minutes. If you don't receive
the email within 15 minutes, check your spam folder.
</context>

Based on the documents above, answer the customer's question.
If the documents don't contain the answer, say so.

User: I forgot my password. How do I reset it?

الخطوة 4: التوليد (Generate)

يقرأ Claude البرومبت المعزز (مع السياق) ويولّد رداً مبنياً على الحقائق المسترجعة. لأن النموذج يملك الحقائق أمامه، لا يستطيع الهلوسة — إما يستخدم المستندات أو يرفض الإجابة.

💡

تشبيه: مساعد باحث

بدون RAG، سؤال Claude يشبه سؤال شخص أن يجيب من ذاكرته فقط. مع RAG، يشبه سؤاله وهو يقرأ المستندات ذات الصلة. يستطيع الاستشهاد بالمصادر ويكون واثقاً من دقة إجابته.

3

التضمينات والبحث المتجهي

سحر RAG يحدث في خطوة الاسترجاع. كيف يعرف النظام أي المستندات ذات صلة؟ الجواب: التضمينات (Embeddings).

ما هي التضمينات؟

التضمين هو تمثيل رقمي للنص. نموذج (مثل text-embedding-3-small من OpenAI) يحوّل النص إلى متجه — قائمة من الأرقام.

Embedding Example 
"How do I reset my password?"
  ↓ (embedding model)
[0.123, -0.456, 0.789, -0.321, ..., 0.654]  ← 384 or 1536 dimensions

التشابه الدلالي

الفكرة الأساسية: النصوص المتشابهة تُنتج تضمينات متشابهة. يمكنك قياس التشابه بحساب المسافة بين المتجهات. كلما اقترب متجهان، كان النصان أكثر تشابهاً.

🔍

كيف يعمل البحث المتجهي

1. حوّل الاستعلام إلى تضمين. 2. ابحث عن أقرب المستندات المجاورة في الفضاء المتجهي. 3. أعد أفضل K نتيجة.

📦

قواعد البيانات المتجهية

قواعد بيانات متخصصة (Pinecone، Weaviate، Chroma، Milvus) تخزن التضمينات وتتيح بحث الجار الأقرب السريع عبر ملايين المستندات.

💡

تشبيه: بطاقات فهرس المكتبة

تخيل فهرس بطاقات المكتبة من عصر ما قبل الرقمنة. كل كتاب له بطاقة فهرس بتصنيفات موضوعية. للعثور على كتب عن "التعلم الآلي"، تبحث في ذلك التصنيف وتحصل على قائمة بالكتب ذات الصلة. التضمينات تعمل بنفس الطريقة — هي "تصنيف موضوعي" رقمي يتيح لك إيجاد المستندات المتشابهة فوراً.

⚠️

جودة نموذج التضمين مهمة

ليست كل نماذج التضمين متساوية في الجودة. استخدام نموذج تضمين ضعيف يعني استرجاعاً سيئاً، مما يعني أنك ستمرر مستندات غير ذات صلة لـ Claude. قم دائماً بقياس جودة الاسترجاع. text-embedding-3 من OpenAI وتضمينات Anthropic خيارات جيدة.

4

استراتيجيات التقسيم

المستندات الخام طويلة جداً لتضمينها بكفاءة. تحتاج لتقسيمها إلى أجزاء (chunks). استراتيجية التقسيم تؤثر بشكل كبير على جودة الاسترجاع.

⚙️

التقسيم بحجم ثابت

الطريقة: قسّم كل N توكن/كلمة. المزايا: بسيط ومتوقع. العيوب: يتجاهل الحدود الدلالية، قد يقسم الجمل.

🧠

التقسيم الدلالي

الطريقة: قسّم بناءً على تغييرات الموضوع وفواصل الفقرات. المزايا: يحافظ على المعنى. العيوب: أكثر تعقيداً، أحجام متغيرة.

📚

التقسيم التكراري

الطريقة: جرّب الحدود الدلالية أولاً، ثم ارجع للحجم الثابت إذا لزم الأمر. المزايا: أفضل ما في العالمين. العيوب: أبطأ في التنفيذ.

🎯

تقسيم خاص بالمجال

الطريقة: احترم البنية (فصول، أقسام، دوال برمجية). المزايا: مثالي للمجال. العيوب: يتطلب معرفة بالمجال.

أفضل الممارسات

  • ابدأ بأجزاء من 512-1024 توكن: هذا الحجم الافتراضي الجيد. الأجزاء الأكبر أقل دقة؛ الأصغر قد تفتقر للسياق.
  • استخدم التداخل: أضف 20-50 توكناً من التداخل بين الأجزاء للحفاظ على السياق عبر الحدود.
  • قم بالقياس: اختبر أحجام أجزاء مختلفة على استعلامات حقيقية وقس دقة الاسترجاع.
  • احفظ البيانات الوصفية: تتبع المستند المصدر والقسم والموقع في المستند الأصلي.
Python — Simple Chunking 
def chunk_text(text, chunk_size=512, overlap=50):
    # Simple fixed-size chunking with overlap
    tokens = text.split()
    chunks = []

    for i in range(0, len(tokens), chunk_size - overlap):
        chunk = ' '.join(tokens[i:i + chunk_size])
        chunks.append(chunk)

    return chunks

# Usage:
doc = "Your long document here..."
chunks = chunk_text(doc, chunk_size=512, overlap=50)

نمط برومبت RAG

كيف تحقن السياق المسترجع في البرومبت؟ هناك نمط مجرّب يعمل بشكل جيد.

Python — RAG Prompt Pattern 
import anthropic

client = anthropic.Anthropic()

# 1. Retrieve relevant documents (from vector DB)
query = "How do I configure API authentication?"
retrieved_docs = retrieve_from_vector_db(query)  # Returns top K docs

# 2. Format context
context = "\n\n---\n\n".join(retrieved_docs)

# 3. Build augmented prompt
SYSTEM_PROMPT = """You are a technical support assistant.
Use the provided documentation to answer questions.
If the docs don't contain the answer, say so clearly.
Never make up information."""

user_message = f"""<context>
{context}
</context>

User Question: {query}"""

# 4. Call Claude with augmented prompt
response = client.messages.create(
    model="claude-opus-4-6",
    max_tokens=1024,
    system=SYSTEM_PROMPT,
    messages=[{"role": "user", "content": user_message}]
)

print(response.content[0].text)

مشاكل RAG الشائعة وحلولها

1. استرجاع غير ذي صلة

⚠️

المشكلة

تسترجع مستندات لا تجيب فعلاً على السؤال. قاعدة البيانات المتجهية تجد مستندات متعلقة بالموضوع لكن ليس الإجابة المحددة.

الحلول: نموذج تضمين أفضل، تقسيم أفضل، زيادة عدد النتائج top-K، بحث هجين (متجهي + كلمات مفتاحية).

2. الضياع في المنتصف

⚠️

المشكلة

عندما تسترجع مستندات كثيرة (+10)، يتجاهل Claude أحياناً معلومات مهمة في المنتصف ويركز على البداية والنهاية.

الحلول: أعد ترتيب المستندات حسب الصلة قبل الحقن، اكتفِ بأفضل 3 إلى 5، أو استخدم نموذجاً أحدث (يتعامل مع السياق الطويل بشكل أفضل).

3. تجاوز السياق

⚠️

المشكلة

تسترجع سياقاً كثيراً بحيث تتجاوز نافذة سياق النموذج أو تجعل الطلب مكلفاً جداً.

الحلول: أحجام أجزاء أصغر، top-K أقل، تقدير حجم السياق قبل استدعاء API، استخدام نماذج أصغر للتحقق.

4. معلومات متناقضة

⚠️

المشكلة

تسترجع مستندات بمعلومات متناقضة. على Claude أن يختار أيها يثق به، وقد لا يختار الصحيح.

الحلول: أضف بيانات وصفية للمصدر (تاريخ المستند، الإصدار)، أخبر Claude أي المصادر موثوقة، تحقق من التناقضات في قاعدة معرفتك.

تطبيق عملي: RAG بسيط ببايثون

لنبنِ نظام RAG كاملاً باستخدام Chroma (قاعدة بيانات متجهية مفتوحة المصدر) وواجهة Anthropic API.

Python — Complete RAG System 
#!/usr/bin/env python3
import anthropic
import chromadb
from chromadb.config import Settings

# Initialize clients
client = anthropic.Anthropic()
chroma_client = chromadb.Client()

# Create a collection (like a table)
collection = chroma_client.create_collection(name="company_docs")

# Add documents to the vector store
documents = [
    "Our API uses OAuth 2.0 for authentication. "
    "Request an access token with your client ID and secret.",

    "Rate limits: 100 requests per minute for free tier, "
    "1000 per minute for premium.",

    "To set up webhooks, go to Settings > Integrations. "
    "All webhook payloads include a signature for verification.",
]

# Chroma automatically embeds documents
collection.add(
    documents=documents,
    ids=["doc1", "doc2", "doc3"]
)

# User query
query = "How do I authenticate with your API?"

# Retrieve relevant documents
results = collection.query(
    query_texts=[query],
    n_results=2  # Top 2 results
)

# Format retrieved docs
context = "\n".join(results["documents"][0])

# Build augmented prompt
system_prompt = """You are a helpful API documentation assistant.
Answer questions using the provided documentation.
If the docs don't address the question, say so."""

user_message = f"""<context>
{context}
</context>

Question: {query}"""

# Call Claude
response = client.messages.create(
    model="claude-opus-4-6",
    max_tokens=500,
    system=system_prompt,
    messages=[{"role": "user", "content": user_message}]
)

print("Answer:", response.content[0].text)

★

تحديث 2025+: أنماط RAG المتقدمة

يتطور RAG بسرعة. إليك نمطين متقدمين يُستخدمان في أنظمة الإنتاج الحديثة.

🕸️ GraphRAG (رسم بياني معرفي)

بدلاً من البحث عن أجزاء نصية، يبني GraphRAG رسماً بيانياً معرفياً يستخلص الكيانات والعلاقات. ممتاز لأسئلة الاستدلال متعدد الخطوات مثل "ما العلاقة بين X و Y؟" حيث يفشل RAG التقليدي.

🤖 RAG الوكيلي (Agentic RAG)

الوكيل يقرر متى وماذا يسترجع بشكل تكراري. يصيغ استعلاماً، يقيّم النتائج، يعيد الصياغة إذا لزم الأمر، ويقرر متى لديه سياق كافٍ للإجابة. يحسّن جودة الاسترجاع بشكل كبير للأسئلة المعقدة.

ملاحظة: بروتوكول سياق النموذج (MCP) يوحّد طريقة اتصال أنظمة RAG بمصادر البيانات المختلفة — قواعد البيانات، واجهات API، أنظمة الملفات — عبر واجهة موحدة بدلاً من كتابة موصلات مخصصة لكل مصدر.

ملخص الموضوع 7

RAG (التوليد المعزز بالاسترجاع) هو أهم نمط في أنظمة الذكاء الاصطناعي الإنتاجية. يحل ثلاث قيود جوهرية للنماذج اللغوية: المعرفة القديمة والهلوسات وعدم الوصول للبيانات الخاصة. المسار بسيط: استعلام ← استرجاع ← تعزيز ← توليد. يستخدم الاسترجاع التضمينات والبحث المتجهي لإيجاد المستندات ذات الصلة. استراتيجية التقسيم تؤثر بشكل كبير على جودة الاسترجاع. المشاكل الشائعة تشمل الاسترجاع غير ذي الصلة وتأثير الضياع في المنتصف وتجاوز السياق والمعلومات المتناقضة — ولكل منها حلول عملية.
