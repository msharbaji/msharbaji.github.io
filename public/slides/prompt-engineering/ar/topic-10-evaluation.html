تقييم جودة البرومبت

قِس وسجّل وحسّن برومبتاتك بشكل منهجي — لأنك لا تستطيع تحسين ما لا تستطيع قياسه.

1

لماذا التقييم مهم

تعتمد فرق كثيرة على "الحدس" للحكم على جودة البرومبت: "هذا يبدو صحيحاً" أو "جربت عدة نسخ وهذه بدت الأفضل." هذا أسلوب قصصي وغير موثوق. في بيئة الإنتاج، تحتاج إلى تقييم منهجي.

⚠️

مشكلة الاختبار القصصي

الاختبار على 5 أمثلة لا يخبرك إن كان البرومبت يعمل على 100,000 مثال. البرومبت الذي "يبدو جيداً" قد يكون هشاً في الواقع. تحتاج إلى بيانات ومقاييس، وليس حدساً.

📈

فوائد التقييم المنهجي

قياس التحسينات: اعرف إن كان التغيير قد ساعد فعلاً. اكتشاف الانحدارات: اكتشف متى تعطّل التحديثات وظائف سابقة. مقارنة النسخ: اختر بين بدائل البرومبت بشكل موضوعي. تتبع الاتجاهات: راقب إن كانت الجودة تتحسن أو تتراجع بمرور الوقت. بناء الثقة: انشر في بيئة الإنتاج وأنت واثق أن البرومبت تم التحقق منه.

✅

المعيار الذهبي

أنشئ مجموعة بيانات اختبار (20-50+ مثالاً) مع المخرجات المتوقعة. شغّل البرومبت على جميع حالات الاختبار. سجّل النتائج تلقائياً أو بتقييم بشري. تتبّع الدرجات بمرور الوقت أثناء التكرار.

2

أنواع التقييم

هناك ثلاث مقاربات رئيسية لتقييم البرومبتات. لكل منها مزايا وعيوب.

👥

التقييم البشري

المزايا: الأكثر دقة، يلتقط الفروق الدقيقة. العيوب: مكلف، بطيء، لا يتوسع لآلاف الأمثلة.

⚙️

المقاييس الآلية

المزايا: سريع، رخيص، قابل للتكرار. العيوب: قد يفوّت مشاكل دقيقة، يتطلب اختيار مقياس جيد.

🤖

نموذج لغوي كحَكَم

المزايا: يفهم الفروق الدقيقة، يتوسع بشكل جيد. العيوب: مكلف، قد يكون متحيزاً بسلوك نموذج الحَكَم.

🔄

هجين

المزايا: الأفضل من جميع العوالم. العيوب: الأكثر تعقيداً في الإعداد.

💡

تشبيه: تقييمات المطاعم

التقييم البشري: ناقد طعام يتذوق الطبق. المقاييس الآلية: تحقق إن كان ساخناً، ودرجة الحموضة متعادلة، وما إلى ذلك. نموذج لغوي كحَكَم: طاهٍ آخر يتذوقه ويعطيه درجة. هجين: مقاييس + ناقد واحد للحالات الحدية.

3

بناء مجموعة بيانات التقييم

أساس التقييم الجيد هو مجموعة بيانات اختبار جيدة. إنشاؤها يستغرق وقتاً لكنه يستحق العناء.

البنية: مدخل ← مخرج متوقع

لكل حالة اختبار، تحتاج مدخلاً (برومبت) والمخرج المتوقع أو المثالي. خزّنها كـ JSON للوصول البرمجي السهل.

توصيات حجم مجموعة البيانات

  • 20-30 مثالاً: الحد الأدنى لاكتشاف المشاكل الكبرى.
  • 50-100 مثال: تغطية جيدة لسيناريوهات مختلفة.
  • 200+: اختبار شامل عبر الفئات والحالات الحدية.
💎

تنوع الأمثلة مهم

لا تختبر فقط المسارات السعيدة. أضف حالات حدية: مدخلات فارغة، مدخلات طويلة جداً، طلبات غامضة، طلبات بلغات مختلفة، وما إلى ذلك. هذا يكتشف الأخطاء قبل وصولها إلى بيئة الإنتاج.

4

نمط النموذج اللغوي كحَكَم

استخدم Claude نفسه للحكم على جودة المخرجات. إنه فعّال بشكل مدهش ويفهم فروقاً دقيقة تفوتها المقاييس البسيطة.

برومبت الحَكَم

اكتب برومبت نظام يصف شكل المخرج الجيد. أعطه المدخل والمخرج للتقييم. اطلب منه التسجيل والشرح.

Python — LLM-as-Judge 
import anthropic
import json

judge_system = """
You are an expert evaluator of code quality.
Score code on:
1. Correctness (does it work?)
2. Readability (is it clear?)
3. Efficiency (is it optimal?)

Return a JSON object with:
- score (1-10)
- reasoning (brief explanation)
- issues (list of problems, or empty)
"""

client = anthropic.Anthropic()

def evaluate_code(code_to_review):
    judge_input = f"""
Evaluate this code:

```python
{code_to_review}
```

Return JSON with score, reasoning, and issues."""

    response = client.messages.create(
        model="claude-opus-4-6",
        max_tokens=500,
        system=judge_system,
        messages=[{
            "role": "user",
            "content": judge_input
        }]
    )

    # Parse the JSON response
    result_text = response.content[0].text
    try:
        result = json.loads(result_text)
        return result["score"]
    except:
        print("Failed to parse judge response")
        return None
⚠️

تحيز نموذج الحَكَم

قد يكون لنموذج الحَكَم تحيز. نماذج مختلفة قد تعطي درجات مختلفة. للتخفيف: استخدم نفس نموذج الحَكَم لجميع التقييمات، كن صريحاً في برومبت الحَكَم، تحقق يدوياً بالتقييم البشري على مجموعة فرعية.

5

المقاييس الآلية

لمهام معينة، تعمل المقاييس الآلية بشكل جيد. إنها سريعة وحتمية. اختر المقياس المناسب لمهمتك.

✓

التطابق الدقيق

هل المخرج == المتوقع؟ ثنائي: نعم/لا. جيد لـ: الترجمة، المخرجات المنظمة. سيء لـ: المهام المفتوحة.

📏

درجة BLEU

تداخل N-gram مع المرجع. معيار للترجمة. النطاق: 0-100. كلما ارتفع كان أفضل.

🎯

درجة ROUGE

موجّه نحو الاستدعاء للتلخيص. يقيس التداخل مع الملخصات المرجعية. جيد لـ: تقييم الملخصات.

📊

تشابه الجيب تمام

تشابه قائم على التضمين. يلتقط المعنى الدلالي. النطاق: 0-1. جيد لـ: النصوص المفتوحة.

Python — Automated Metrics 
from difflib import SequenceMatcher
import numpy as np

# Exact Match
def exact_match(output, expected):
    return 1.0 if output.strip() == expected.strip() else 0.0

# Sequence Similarity (for closeness)
def similarity_score(output, expected):
    return SequenceMatcher(None, output, expected).ratio()

# Usage:
output = "The Great Gatsby is about wealth and love in the 1920s"
expected = "A novel exploring American Dream and 1920s wealth"

score1 = exact_match(output, expected)  # 0.0
score2 = similarity_score(output, expected)  # 0.42
✅

اختيار المقياس

استخدم التطابق الدقيق للمهام الحتمية (الكود، الترجمة، الرياضيات). استخدم ROUGE/BLEU للتلخيص. استخدم تشابه الجيب تمام أو النموذج اللغوي كحَكَم للمهام المفتوحة حيث توجد إجابات صحيحة متعددة.

بناء خط أنابيب التقييم

إليك خط أنابيب تقييم كامل يختبر البرومبت ويولّد تقرير درجات.

Python — Complete Eval Pipeline 
import anthropic
import json
from dataclasses import dataclass

client = anthropic.Anthropic()

@dataclass
class TestCase:
    id: str
    input: str
    expected_output: str

SYSTEM_PROMPT = """You are a code reviewer..."""

def evaluate_single_case(test_case: TestCase) -> dict:
    # Get model output
    response = client.messages.create(
        model="claude-opus-4-6",
        max_tokens=500,
        system=SYSTEM_PROMPT,
        messages=[{
            "role": "user",
            "content": test_case.input
        }]
    )
    actual_output = response.content[0].text

    # Score using LLM-as-judge
    judge_prompt = f"""
Input: {test_case.input}
Expected: {test_case.expected_output}
Actual: {actual_output}

Score accuracy 1-10."""

    judge_response = client.messages.create(
        model="claude-opus-4-6",
        max_tokens=100,
        messages=[{
            "role": "user",
            "content": judge_prompt
        }]
    )
    judge_output = judge_response.content[0].text

    return {
        "test_id": test_case.id,
        "actual_output": actual_output,
        "judge_score": judge_output
    }

def run_eval_pipeline(test_cases: list[TestCase]) -> dict:
    results = []
    scores = []

    for test_case in test_cases:
        result = evaluate_single_case(test_case)
        results.append(result)
        # Extract numeric score (simplified)
        score = int(result["judge_score"][0]) if result["judge_score"][0].isdigit() else 5
        scores.append(score)

    return {
        "total_tests": len(test_cases),
        "average_score": sum(scores) / len(scores),
        "min_score": min(scores),
        "max_score": max(scores),
        "results": results
    }

# Usage:
test_data = [
    TestCase("1", "Review: def x(): pass", "Good but lacks docstring"),
    # ...
]
report = run_eval_pipeline(test_data)
print(json.dumps(report, indent=2))

التحسين المستمر

التقييم مفيد فقط إذا استخدمت النتائج فعلاً للتحسين. إليك سير العمل.

حلقة التحسين

  1. خط الأساس: قيّم البرومبت الحالي. سجّل درجة خط الأساس.
  2. التكرار: أجرِ تحسينات مستهدفة (برومبت نظام أفضل، أمثلة قليلة، إلخ).
  3. الاختبار: شغّل خط أنابيب التقييم الكامل على النسخة الجديدة.
  4. المقارنة: هل الدرجة أفضل؟ إذا نعم، احتفظ بها. إذا لا، تراجع أو جرّب بطريقة مختلفة.
  5. التكرار: عد إلى الخطوة 2.

اختبار الانحدار

احتفظ بمجموعة بيانات التقييم وأعد تشغيلها كلما حدّثت البرومبت. هذا يكتشف الآثار الجانبية غير المقصودة حيث يُصلح تغيير بعض الحالات لكنه يعطّل أخرى.

Python — Track Improvements 
scores = {
    "v1_baseline": 65,
    "v2_better_system_prompt": 72,
    "v3_added_examples": 78,
    "v4_clarified_format": 81
}

for version, score in scores.items():
    improvement = score - 65
    print(f"{version}: {score}/100 (+{improvement})")

CI/CD للبرومبتات

بعض الفرق تدمج خطوط أنابيب التقييم في CI/CD. كل commit للبرومبت يطلق جولة تقييم. إذا انخفضت الدرجة عن حد معين، يُحظر الـ commit. هذا يمنع الانحدارات من الوصول إلى بيئة الإنتاج.

ملخص الموضوع 12

التقييم المنهجي يتفوق على الاختبار القصصي. أنشئ مجموعة بيانات تقييم بـ 50+ حالة اختبار (مدخل + مخرج متوقع). اختر طرق التقييم: التقييم البشري (الأكثر دقة)، المقاييس الآلية (سريع)، النموذج اللغوي كحَكَم (قابل للتوسع). استخدم المقياس المناسب لمهمتك: التطابق الدقيق للكود/الترجمة، ROUGE للملخصات، تشابه الجيب تمام للمهام المفتوحة. نفّذ خط أنابيب يختبر البرومبت ويسجّله تلقائياً. استخدم النتائج للتحسين التكراري. تتبّع الدرجات عبر النسخ لقياس التقدم. شغّل اختبارات الانحدار كلما حدّثت لاكتشاف الآثار الجانبية غير المقصودة.
