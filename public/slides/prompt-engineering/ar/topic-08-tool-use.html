استخدام الأدوات واستدعاء الدوال

اجعل النموذج اللغوي يتخذ إجراءات في العالم الحقيقي — استدعاء واجهات برمجية، وتشغيل أكواد، والوصول لبيانات حية.

1

ما هو استخدام الأدوات؟

بشكل افتراضي، النماذج اللغوية الكبيرة يمكنها فقط توليد نصوص. استخدام الأدوات (المعروف أيضاً باستدعاء الدوال) يتيح للنموذج أن يفعل شيئاً يقرر هو فعله — وليس شيئاً تطلب منه فعله أنت. النموذج يختار متى وكيف يستخدم الأدوات بناءً على طلب المستخدم.

✅

فكرة جوهرية: النموذج يقرر

الفرق الجوهري عن RAG: في RAG، أنت تسترجع المستندات وتمررها. أما في استخدام الأدوات، النموذج يحلل طلب المستخدم ويقرر ما إذا كانت هناك حاجة لأداة، وأي أداة يستدعيها وبأي معاملات.

🤔

اتخاذ القرار

النموذج يحلل الطلب ويقرر إذا كان استدعاء أداة ضرورياً. يمكنه رفض استخدام الأدوات حتى لو كانت متاحة.

🎯

اختيار المعاملات

النموذج يختار المعاملات للأداة. "ابحث عن X" والنموذج يملأ X بناءً على السياق.

🔄

استدلال متعدد الخطوات

النموذج يمكنه استدعاء عدة أدوات بالتتابع، مستخدماً مخرجات أداة كمدخلات للأداة التالية.

🌍

تأثير حقيقي

استدعاءات الأدوات يمكن أن يكون لها آثار جانبية: إجراء عمليات شراء، إرسال بريد إلكتروني، تحديث قواعد بيانات. يتطلب اعتبارات أمان.

💡

تشبيه: مساعد ذكي

بدون استخدام الأدوات: تسأل سؤالاً وتحصل على إجابة من الذاكرة. مع استخدام الأدوات: تسأل مساعداً يمكنه التقاط الهاتف (استدعاء API)، والتحقق من خزانة الملفات (استعلام قاعدة البيانات)، أو إجراء حساب (تشغيل كود) ليعطيك إجابة دقيقة. المساعد يقرر أي أدوات يستخدم.

الخطوة 1: تعريف الأدوات

تصف الأدوات المتاحة بصيغة JSON schema. تخبر النموذج بما تفعله كل أداة، وما المعاملات التي تقبلها، وما الذي تُرجعه. النموذج يقرأ هذه التعريفات ويستخدمها ليقرر متى يستدعي كل أداة.

الخطوة 2: النموذج يختار

عندما ترسل رسالة المستخدم، يحللها Claude مقابل تعريفات الأدوات. إذا رأى أن أداة ما ستساعد، يُرجع استجابة استخدام أداة بدلاً من نص. يحدد: أي أداة، وبأي معاملات.

الخطوة 3: أنت تنفذ

الكود الخاص بك يستقبل استدعاء الأداة، يتحقق من المعاملات، ينفذها (يستدعي API، يستعلم قاعدة البيانات، يشغّل الدالة)، ويحصل على نتيجة. أنت المسؤول عن التنفيذ الفعلي.

الخطوة 4: النموذج يستجيب

تمرر نتيجة الأداة مرة أخرى إلى Claude في المحادثة. الآن لدى Claude بيانات حقيقية ويمكنه توليد استجابة مدروسة. قد يؤدي هذا إلى استدعاءات أدوات إضافية، أو يمكن لـ Claude الإجابة مباشرة على المستخدم.

⚠️

حاسم: النموذج لا ينفذ الأدوات

النموذج لا يستدعي أبداً أدواتك فعلياً. هو فقط يطلب منك استدعاءها. يجب عليك أنت التعامل مع التنفيذ، والتحقق من الأخطاء، والتحقق الأمني. هذه ميزة وليست عيباً — تبقي النموذج آمناً وتبقيك أنت في موضع السيطرة.

تعريف الأدوات (صيغة Anthropic)

أدوات Anthropic تستخدم JSON schema واضح. إليك كيفية تعريفها.

JSON — Tool Definition 
{
  "name": "get_weather",
  "description": "Get current weather for a city",
  "input_schema": {
    "type": "object",
    "properties": {
      "city": {
        "type": "string",
        "description": "City name (e.g., 'San Francisco')"
      },
      "unit": {
        "type": "string",
        "enum": ["celsius", "fahrenheit"],
        "description": "Temperature unit"
      }
    },
    "required": ["city"]
  }
}

مثال كامل: عدة أدوات

Python — Define and Use Tools 
import anthropic
import json

client = anthropic.Anthropic()

# Define available tools
tools = [
    {
        "name": "get_weather",
        "description": "Get current weather for a location",
        "input_schema": {
            "type": "object",
            "properties": {
                "city": {"type": "string",
                            "description": "City name"},
                "unit": {"type": "string",
                          "enum": ["celsius", "fahrenheit"]}
            },
            "required": ["city"]
        }
    },
    {
        "name": "calculate",
        "description": "Perform a calculation",
        "input_schema": {
            "type": "object",
            "properties": {
                "expression": {"type": "string",
                                  "description": "Math expression (e.g., '2+2*3')"}
            },
            "required": ["expression"]
        }
    }
]

# Make API call with tools
response = client.messages.create(
    model="claude-opus-4-6",
    max_tokens=1024,
    tools=tools,
    messages=[
        {"role": "user", "content": "What's the weather in Paris and what is 15 * 3?"}
    ]
)

      "properties": { ... },

متى تستخدم كل صيغة

أدوات Anthropic: صيغة نظيفة وبسيطة. دعم ممتاز للتوثيق. صيغة OpenAI: مدعومة على نطاق أوسع. استخدمها إذا كنت تستخدم نماذج OpenAI أو تحتاج توافقية أوسع. للمشاريع الجديدة، صيغة Anthropic أبسط وأقل عرضة للأخطاء.

حلقة استخدام الأدوات

التعامل مع استجابات الأدوات يتطلب حلقة. النموذج قد يستدعي أدوات، يحصل على نتائج، ويستدعي أدوات أخرى.

Python — Complete Tool Loop 
def process_tool_call(tool_name, tool_input):
    """Execute a tool and return result"""
    if tool_name == "get_weather":
        return f"Weather in {tool_input['city']}: Sunny, 72°F"
    elif tool_name == "calculate":
        return f"Result: {eval(tool_input['expression'])}"
    else:
        return "Unknown tool"

# Main loop
messages = [{"role": "user", "content": "What's weather in NYC and compute 10+5?"}]

while True:
    # Call Claude with tools
    response = client.messages.create(
        model="claude-opus-4-6",
        max_tokens=1024,
        tools=tools,
        messages=messages
    )

    # Check if we need to handle tool calls
    if response.stop_reason == "tool_use":
        # Process each tool call
        tool_calls = [block for block in response.content
                      if block.type == "tool_use"]

        # Add assistant response to messages
        messages.append({"role": "assistant", "content": response.content})

        # Execute tools and collect results
        for tool_call in tool_calls:
            result = process_tool_call(tool_call.name, tool_call.input)

            # Add tool result to messages
            messages.append({
                "role": "user",
                "content": [{
                    "type": "tool_result",
                    "tool_use_id": tool_call.id,
                    "content": result
                }]
            })
    else:
        # Model generated final response (no more tool calls)
        final_response = response.content[0].text
        print(final_response)
        break

نمط الحلقة

1. استدعِ Claude مع الأدوات في قائمة الرسائل. 2. إذا كانت الاستجابة "tool_use"، عالج استدعاءات الأدوات. 3. أضف استجابة المساعد ونتائج الأدوات مرة أخرى إلى الرسائل. 4. عُد للخطوة 1. 5. عندما تكون الاستجابة "end_turn" (أو لا توجد استدعاءات أدوات)، لديك الإجابة النهائية.

معالجة الأخطاء والأمان

ماذا يحدث عندما تفشل الأدوات؟

الأدوات ستفشل. انقطاع الشبكة، مدخلات غير صالحة، تجاوز حدود API، أخطاء الصلاحيات. يجب عليك:

  • التعامل مع الأخطاء بسلاسة: لا تتوقف. أرجع رسالة خطأ إلى Claude.
  • التحقق من المدخلات: تحقق من معاملات الأداة قبل التنفيذ. أخبر Claude إذا كان معامل غير صالح.
  • تحديد مهلة زمنية: يجب أن تنتهي مهلة الأدوات بعد فترة معقولة، لا أن تعلق للأبد.
  • تسجيل كل شيء: لأغراض التصحيح، سجّل ما طلبه النموذج وما حدث.
Python — Error Handling 
def process_tool_call(tool_name, tool_input):
    try:
        if tool_name == "web_search":
            # Validate input
            query = tool_input.get("query", "")
            if not query or len(query) > 500:
                return "Error: query required, max 500 chars"

            # Execute with timeout
            result = search_api.search(query, timeout=5)

    except TimeoutError:

الأمان وحواجز الحماية

⚠️

الأدوات يمكن أن يكون لها تأثير حقيقي

أداة تحذف ملفات، أو تحوّل أموالاً، أو ترسل بريداً إلكترونياً يمكن أن تسبب ضرراً حقيقياً إذا أُسيء استخدامها. نفّذ دائماً حواجز حماية: تدفقات الموافقة للإجراءات الحساسة، تحديد المعدل، سجلات التدقيق، وفحوصات الصلاحيات بناءً على هوية المستخدم.

💎

تدفقات الموافقة

للإجراءات الحساسة (المدفوعات، الحذف، استدعاءات API ذات الآثار الجانبية)، فكّر في طلب موافقة بشرية. النموذج يمكنه طلب إجراء، لكن الكود الخاص بك يسأل المستخدم عن التأكيد قبل التنفيذ.

★

تحديث 2025+: بروتوكول سياق النموذج (MCP)

في 2025، تبنّت الصناعة معياراً موحداً لربط النماذج بالأدوات: بروتوكول سياق النموذج (MCP). لم يعد عليك كتابة تعريفات أدوات مخصصة لكل نموذج.

🌐 معيار عالمي

MCP مدعوم من Anthropic وOpenAI وGoogle وMicrosoft. يعمل عبر جميع مزوّدي النماذج بتعريف أداة واحد موحد — اكتب مرة واحدة، يعمل في كل مكان.

🔌 خوادم MCP

كل أداة تعمل كـخادم MCP — خدمة خفيفة تصف قدراتها ومدخلاتها ومخرجاتها. النموذج يكتشف الأدوات تلقائياً ويستخدمها بدون إعداد مخصص.

📦 نظام بيئي غني

آلاف خوادم MCP الجاهزة متاحة: قواعد البيانات، GitHub، Slack، أنظمة الملفات، واجهات API والمزيد. استخدم خوادم جاهزة بدلاً من بناء موصلاتك الخاصة.

🔒 أمان مدمج

يتضمن MCP المصادقة والتفويض وتحديد النطاق بشكل مدمج. تحكّم بدقة في ما يمكن لكل نموذج الوصول إليه وتنفيذه.

MCP مقابل تعريفات الأدوات المخصصة: تعريفات الأدوات المخصصة (التي تعلمتها سابقاً) لا تزال مفيدة للأدوات الداخلية الخاصة. لكن للأدوات الشائعة (قواعد البيانات، API، خدمات الطرف الثالث)، استخدم خوادم MCP الجاهزة — أسرع في الإعداد وأكثر أماناً.

ملخص الموضوع 8

استخدام الأدوات (استدعاء الدوال) يتيح للنماذج اتخاذ إجراءات في العالم الحقيقي عبر استدعاء واجهات برمجية، وتشغيل أكواد، والوصول لبيانات حية. النموذج يقرر متى وكيف يستخدم الأدوات بناءً على طلبات المستخدم. خط الأنابيب هو: تعريف الأدوات ← النموذج يختار ← أنت تنفذ ← النموذج يستجيب. يجب عليك تنفيذ الحلقة: استدعِ Claude، تحقق من طلبات الأدوات، نفّذ الأدوات، أرجع النتائج، كرر حتى الانتهاء. الأدوات يمكنها دمج البحث في الويب، والآلات الحاسبة، وقواعد البيانات، وواجهات API، وعمليات الملفات، وتنفيذ الأكواد. حاسم: أنت تنفذ الأدوات، وليس النموذج. أنت المسؤول عن التحقق ومعالجة الأخطاء والأمان.
