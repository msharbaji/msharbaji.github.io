البرومبت المتقدم

المخرجات المنظمة، وسوم XML، والفواصل، وأنماط برومبت النظام المتقدمة للحصول على نتائج بجودة الإنتاج.

1

وسوم XML للهيكلة

يستجيب Claude (من Anthropic) بشكل استثنائي لـالبرومبتات الموسومة بـ XML. تغليف الأقسام في وسوم مثل <context> و<instructions> و<output_format> يساعد النموذج على تحليل الطلبات المعقدة بدقة. وسوم XML هي أفضل ممارسة للبرومبتات الإنتاجية.

لماذا تعمل وسوم XML بفعالية:

  • بنية دلالية واضحة — يفهم النموذج ما يعنيه كل قسم
  • تسلسل هرمي متداخل — يمكن لـ XML أن يحتوي على معلومات متداخلة معقدة
  • حدود لا لبس فيها — لا خلط حول أين يبدأ كل قسم وأين ينتهي
  • مثبت في الإنتاج — يُستخدم على نطاق واسع من قِبل فرق تستخدم Claude في تطبيقات المؤسسات

وسوم XML الشائعة:

📋

<context>

المعلومات الأساسية أو البيانات أو الموقف الذي يحتاج النموذج لفهمه

🎯

<instructions>

ما تريد من النموذج أن يفعله

📦

<output_format>

البنية الدقيقة للرد (JSON، XML، markdown، إلخ.)

✅

<rules>

القيود والحواجز ("لا تفعل X"، "افعل دائماً Y")

💡

<examples>

أمثلة توضيحية (few-shot) تُظهر السلوك المتوقع

🎭

<system_role>

هوية النموذج وخبرته

Prompt — XML Structure 
<system_role>
You are a senior Python architect.
</system_role>

<context>
The user has a function that processes
large lists (100k+ items). It's slow.
</context>

<instructions>
Analyze the code and suggest optimizations.
Focus on algorithm complexity first, then
memory usage, then parallelization.
</instructions>

<rules>
- Don't suggest rewrites unless necessary
- Always explain the performance impact
- Cite specific lines of code
- Don't suggest external dependencies
</rules>

<output_format>
{
  "summary": "Brief overview",
  "optimizations": [
    {
      "issue": "What's slow",
      "fix": "How to fix it",
      "performance_gain": "Expected improvement",
      "code_location": "Line X-Y"
    }
  ]
}
</output_format>
💡

نصيحة احترافية: تداخل الوسوم

يمكنك تداخل الوسوم لإنشاء تسلسل هرمي. مثال: <examples><example id="1">...</example></examples>. هذا يساعد في تنظيم البرومبتات المعقدة إلى أقسام منطقية.

2

الفواصل والمحددات

الفواصل (---، ```، ###) تفصل الأقسام بصرياً وتساعد النموذج على فهم الحدود. وهي مفيدة بشكل خاص عندما يحتوي البرومبت على محتوى مقدَّم من المستخدم قد يتضمن نصاً يشبه التعليمات (دفاع ضد حقن البرومبت).

الفواصل الشائعة:

--- (Triple Dash)
Instruction text here.

---

User-provided content here.
This could contain anything.
``` (Triple Backticks)
Instruction text here.

```
User-provided content here.
This could contain anything.
```
### (Markdown Header)
Instruction text here.

### User Content

User-provided content here.
This could contain anything.
== (Double Equal)
Instruction text here.

====================

User-provided content here.
This could contain anything.

متى تستخدم أيّاً منها:

  • --- الأفضل لفصل الأقسام الرئيسية؛ نظيف وسهل القراءة
  • ``` الأفضل للكود أو البيانات التقنية؛ يشير دلالياً إلى "كتلة كود"
  • ### الأفضل للبرومبتات بتنسيق markdown؛ يندمج مع بنية markdown
  • == جيد للتأكيد؛ يبرز بصرياً
✅

فاصل + XML = أقصى وضوح

ادمج الفواصل مع وسوم XML لتحقيق وضوح مطلق: <instructions>...</instructions> --- <user_content>...</user_content>

3

المخرجات المنظمة (JSON)

في التطبيقات الإنتاجية، غالباً ما تحتاج إلى مخرجات JSON مضمونة يمكنك تحليلها برمجياً. يتطلب ذلك أنماط برومبت نظام ومواصفات output_format واضحة. Claude موثوق بدرجة عالية في إنتاج JSON صالح عند الطلب صراحةً.

Prompt — JSON Output Pattern 
<instructions>
You are a sentiment analysis system.
Analyze the text and respond with JSON
containing sentiment, confidence, and
supporting evidence.
</instructions>

<output_format>
{
  "sentiment": "positive|negative|neutral",
  "confidence": 0.0 to 1.0,
  "supporting_quote": "...",
  "explanation": "..."
}

IMPORTANT: Return ONLY valid JSON.
No markdown, no extra text. Just JSON.
</output_format>

Text to analyze: "I love this product!"

كود بايثون لتحليل JSON بأمان:

Python — Safe JSON Parsing 
import json
import anthropic

client = anthropic.Anthropic()

response = client.messages.create(
    model="claude-sonnet-4-5-20250929",
    max_tokens=500,
    messages=[{"role": "user", "content": "..."}]
)

text = response.content[0].text

try:
    result = json.loads(text)
    print(f"Sentiment: {result['sentiment']}")
except json.JSONDecodeError as e:
    print(f"Failed to parse JSON: {e}")
    print(f"Raw response: {text}")
⚠️

تحقق دائماً من صحة JSON

حتى مع التعليمات الصريحة، اغلف تحليل JSON دائماً في كتل try/except. النماذج موثوقة لكنها ليست مثالية. التعامل الرشيق مع الأخطاء ضروري في الإنتاج.

4

قوالب المخرجات

قوالب المخرجات تحدد مسبقاً الشكل الدقيق للرد. بدلاً من القول ببساطة "أعطني ملخصاً"، تقدم البنية الدقيقة التي تريد ملأها. هذا يضمن الاتساق عبر طلبات متعددة.

مثال: قالب ملخص اجتماع

Prompt — Template-Driven Output 
<instructions>
Summarize the meeting transcript using
this exact template. Fill in each section.
</instructions>

Meeting Summary Template:
──────────────────────────
Title: [Meeting name]
Date: [Date]
Duration: [Length in minutes]

Attendees: [List names]

Key Decisions:
1. [Decision 1]
2. [Decision 2]
3. [Decision 3]

Action Items:
- [ ] [Action 1] - Owner: [Name]
- [ ] [Action 2] - Owner: [Name]
- [ ] [Action 3] - Owner: [Name]

Next Steps: [What happens next]

Follow-up Meeting: [Date/Time if scheduled]
──────────────────────────

Now summarize this transcript: ...

فوائد القوالب:

  • الاتساق: كل مخرج يتبع نفس البنية
  • التحليل: سهولة استخراج الحقول من المخرجات القالبية
  • الاكتمال: يضمن عدم تفويت أي قسم
  • الجودة: البنية الصريحة غالباً ما تحسّن الاستدلال
✅

قالب + JSON = تركيبة قوية

ادمج القوالب مع JSON للحصول على أقصى بنية: اجعل القالب يحدد أسماء الحقول، ثم اطلب مخرجات JSON. هذا يمنحك بنية مقروءة للإنسان وتنسيقاً قابلاً للتحليل الآلي في آن واحد.

5

البرومبت السلبي

البرومبت السلبي يخبر النموذج بما لا يجب فعله. رغم أنه ليس بقوة التعليمات الإيجابية، إلا أنه مفيد كحواجز حماية. كن محدداً: "لا تعتذر" تعمل أفضل من "كن واثقاً".

البرومبت السلبي الجيد مقابل السيئ:

❌ سلبي غامض
Don't make mistakes.
Don't be rude.
Don't be boring.
✅ سلبي محدد
Don't include disclaimers or apologies.
Don't use corporate jargon.
Don't make up facts you're unsure about.
Don't exceed 200 words.

أنماط البرومبت السلبي:

🚫

قيود التنسيق

"لا تستخدم markdown أبداً. أعد دائماً نصاً عادياً."

🚫

قيود المحتوى

"لا تذكر المنافسين. لا تكشف تفاصيل داخلية."

🚫

قيود السلوك

"لا تطرح أسئلة متابعة. لا تتردد في إجابتك."

🚫

قيود النبرة

"لا تعتذر. لا تكن رسمياً بشكل مبالغ. لا تستخدم السخرية."

⚠️

فضّل التعليمات الإيجابية

التعليمات السلبية مفيدة لكنها أقل فعالية من الإيجابية. بدلاً من "لا تكن فظاً"، قل "كن ودوداً ومحترماً". القيود السلبية يجب أن تعزز السلوك الإيجابي، لا أن تحل محله.

6

سلسلة البرومبتات

سلسلة البرومبتات (Prompt Chaining) تقسّم المهام المعقدة إلى برومبتات متتابعة. بدلاً من سؤال نموذج واحد لفعل كل شيء، تمرر المخرجات عبر خط أنابيب: المهمة أ ← المهمة ب ← المهمة ج. هذا يحسّن الدقة ويتيح معالجة الأخطاء بين الخطوات.

مثال على السلسلة: تحليل ← استخراج ← تلخيص

Python — Prompt Chaining 
import anthropic
import json

client = anthropic.Anthropic()

def chain_analysis(document):
    # Step 1: Extract key entities
    step1 = client.messages.create(
        model="claude-sonnet-4-5-20250929",
        max_tokens=500,
        messages=[{
            "role": "user",
            "content": f"Extract names, dates, and amounts from this document:\n{document}\n\nReturn JSON."
        }]
    )
    entities = json.loads(step1.content[0].text)

    # Step 2: Analyze sentiment of entities
    step2 = client.messages.create(
        model="claude-sonnet-4-5-20250929",
        max_tokens=500,
        messages=[{
            "role": "user",
            "content": f"Analyze sentiment of these names in context:\n{json.dumps(entities)}\n\nReturn JSON."
        }]
    )
    analysis = json.loads(step2.content[0].text)

    # Step 3: Generate summary
    step3 = client.messages.create(
        model="claude-sonnet-4-5-20250929",
        max_tokens=300,
        messages=[{
            "role": "user",
            "content": f"Summarize this analysis:\n{json.dumps(analysis)}\n\nKeep it under 100 words."
        }]
    )
    summary = step3.content[0].text

    return {
        "entities": entities,
        "analysis": analysis,
        "summary": summary
    }

result = chain_analysis("Alice sold 100 shares...")
print(json.dumps(result, indent=2))

متى تستخدم سلسلة البرومبتات:

  • المهام المعقدة التي تتحلل بطبيعتها إلى خطوات
  • عندما تحتاج إلى التحقق من النتائج الوسيطة
  • عندما تستفيد خطوات مختلفة من برومبتات نظام أو قيم حرارة مختلفة
  • عندما تريد تطبيق قواعد مختلفة على خطوات مختلفة
💡

معالجة الأخطاء في السلاسل

تحقق دائماً من صحة مخرجات كل خطوة قبل تمريرها للخطوة التالية. إذا فشلت الخطوة 2 في إنتاج JSON صالح، يمكنك إعادة المحاولة أو استخدام بديل أو إيقاف السلسلة بشكل رشيق.

البرومبت الفوقي

البرومبت الفوقي (Meta-Prompting) يعني استخدام نموذج لغوي لتوليد أو تحسين البرومبتات. يمكنك أن تطلب من Claude كتابة برومبتات لنفسه أو لنماذج أخرى. هذا مفيد لبناء النماذج الأولية وتوسيع تطوير البرومبتات واستكشاف الصيغ البديلة.

مثال: اطلب من Claude كتابة برومبت

Prompt — Meta-Prompting 
<instructions>
Write a prompt that will make Claude classify
customer support tickets as urgent, normal,
or low-priority. The prompt should:
- Use XML tags for clarity
- Include 3 few-shot examples
- Define output format as JSON
- Include guardrails against bias
</instructions>

Return the complete prompt as a markdown code
block with proper formatting.

كود بايثون: توليد البرومبتات ديناميكياً

Python — Dynamic Prompt Generation 
import anthropic

client = anthropic.Anthropic()

def generate_prompt(task_description):
    """Use Claude to write a prompt."""
    response = client.messages.create(
        model="claude-sonnet-4-5-20250929",
        max_tokens=1000,
        messages=[{
            "role": "user",
            "content": f"""Write a production-grade prompt for this task:

{task_description}

The prompt should use XML tags, include
few-shot examples, define output format,
and include guardrails. Return just the
prompt text, ready to use."""
        }]
    )
    return response.content[0].text

# Generate a prompt for data extraction
task = "Extract invoice data (vendor, amount, date) from unstructured text"
prompt = generate_prompt(task)
print(prompt)

# Now use the generated prompt
response = client.messages.create(
    model="claude-sonnet-4-5-20250929",
    max_tokens=500,
    messages=[{
        "role": "user",
        "content": prompt
    }]
)
print(response.content[0].text)
✅

حالات استخدام البرومبت الفوقي

استخدم البرومبت الفوقي لـ: توليد صيغ متعددة لبرومبت، وتحويل برومبت من تنسيق لآخر، وتوسيع برومبت بسيط إلى نسخة بجودة الإنتاج، أو استكشاف صياغات بديلة.

اختبر فهمك

اختبار سريع — 4 أسئلة

1. لماذا يستجيب Claude بشكل جيد لوسوم XML في البرومبتات؟

2. ما الاستخدام الأساسي لسلسلة البرومبتات؟

3. عند استخدام مخرجات JSON المنظمة، ماذا يجب أن تفعل دائماً؟

4. ما الميزة الرئيسية لقوالب المخرجات؟

✓

ملخص الموضوع 4

إليك ما تعلمته:

وسوم XML تهيكل البرومبتات للوضوح. الفواصل تفصل الأقسام وتحمي من الحقن. مخرجات JSON تجعل الردود قابلة للتحليل الآلي. القوالب تفرض الاتساق. البرومبت السلبي يضيف حواجز حماية. سلسلة البرومبتات تقسّم التعقيد إلى خطوات. البرومبت الفوقي يستخدم النماذج اللغوية لتوليد البرومبتات. هذه التقنيات تتراكم معاً لتحقيق موثوقية بجودة الإنتاج.
