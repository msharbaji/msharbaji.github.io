Async/Await

Rust الـ Async مع Tokio, Futures وAsync Functions وتنفيذ Concurrent Tasks.

1

ما هي Async Programming؟

Async Programming تتيح كتابة كود غير محجوب يمكنه التعامل مع العديد من العمليات المتزامنة بعدد أقل من Threads. وهي مثالية للمهام المرتبطة بالإدخال/الإخراج مثل طلبات الشبكة وقراءة الملفات واستعلامات قواعد البيانات حيث يقضي البرنامج معظم وقته في الانتظار.

Rust
use std::future::Future;
use std::pin::Pin;
use std::task::{Context, Poll};
// A simple future that completes after one poll
struct SimpleFuture {
    done: bool,
impl Future for SimpleFuture {
    type Output = String;
    fn poll(
        mut self: Pin<&mut Self>,
        _cx: &mut Context<'_>,
    ) -> Poll<Self::Output> {
        if self.done {
            Poll::Ready(String::from("Done!"))
        } else {
            self.done = true;
            Poll::Pending
        }
    }
}
💡

Futures كسولة

Future لا يفعل شيئًا حتى يتم استقصاؤه. Function poll() تُرجع Poll::Pending عندما لم تكتمل العملية بعد، و Poll::Ready(value) عندما تكتمل. صياغة async/await تخفي هذه الآلية.

2

Futures وصياغة async/await

صياغة async/await تجعل كتابة الكود الـ Async تبدو مثل الكود المتزامن. Function async fn تُرجع impl Future، و .await تعلّق التنفيذ حتى يكتمل العقد الآجل.

Rust
use std::time::Duration;
use std::thread;
// Without async: verbose callback style
fn fetch_data_callback<F: Fn(String)>(callback: F) {
    thread::spawn(move || {
        thread::sleep(Duration::from_secs(1));
        callback("Data from server".to_string());
    });
// With async: clean and readable
async fn fetch_data_async() -> String {
    println!("Fetching data...");
    "Data from server".to_string()
}
Rust
// Async functions return impl Future
async fn example() {
    println!("Start");
    let result = "completed".to_string();
    println!("Result: {}", result);
#[tokio::main]
async fn main() {
    example().await;
    println!("Async main completed");
}
✨

async fn هي سكر نحوي

كتابة async fn foo() -> i32 مكافئة لكتابة fn foo() -> impl Future<Output = i32>. الـ Compiler يحوّل جسم الـ Function إلى آلة حالة تنفّذ سمة Future.

3

أساسيات وقت تشغيل Tokio

Tokio هو Runtime الـ Async الأكثر شعبية لـ Rust، ويوفر إنشاء المهام والمؤقتات وأدوات الإدخال/الإخراج والمزيد. تحتاج إلى وقت تشغيل لتنفيذ Futures, Standard Library في Rust لا تتضمن واحدًا.

Rust
// Using the #[tokio::main] macro
#[tokio::main]
async fn main() {
    println!("Tokio runtime is running");
    // Spawn an async task
    let handle = tokio::spawn(async {
        println!("Task running");
        "Result".to_string()
    });
    // Wait for task completion
    match handle.await {
        Ok(result) => println!("Got: {}", result),
        Err(_) => println!("Task panicked"),
    }
}
Rust
use tokio::runtime::Runtime;
// Manual runtime creation (useful in non-async main)
fn main() {
    let rt = Runtime::new().unwrap();
    rt.block_on(async {
        println!("Running in async context");
        let result = async_operation().await;
        println!("Result: {}", result);
    });
async fn async_operation() -> i32 {
    42
}
💡

إعداد Cargo.toml

أضف tokio = { version = "1", features = ["full"] } إلى [dependencies]. ميزة "full" تفعّل جميع مكونات Tokio بما في ذلك المؤقتات والإدخال/الإخراج وRuntime متعدد Threads.

4

إنشاء المهام الـ Asyncة

tokio::spawn ينفّذ عقدًا آجلاً بشكل متزامن على وقت تشغيل Tokio. كل مهمة مُنشأة هي وحدة عمل مستقلة يجدولها Runtime بشكل تعاوني.

Rust
#[tokio::main]
async fn main() {
    let mut handles = vec![];
    for i in 0..5 {
        let handle = tokio::spawn(async move {
            println!("Task {} starting", i);
            tokio::time::sleep(
                tokio::time::Duration::from_millis(100 * i as u64)
            ).await;
            println!("Task {} done", i);
            i * 2
        });
        handles.push(handle);
    }
    // Wait for all tasks
    for handle in handles {
        match handle.await {
            Ok(result) => println!("Result: {}", result),
            Err(_) => println!("Task panicked"),
        }
    }
}
Rust
// Tracking concurrent tasks with AtomicUsize
use std::sync::Arc;
use std::sync::atomic::{AtomicUsize, Ordering};
#[tokio::main]
async fn main() {
    let counter = Arc::new(AtomicUsize::new(0));
    for _ in 0..10 {
        let counter = counter.clone();
        tokio::spawn(async move {
            counter.fetch_add(1, Ordering::Relaxed);
            tokio::time::sleep(
                tokio::time::Duration::from_millis(100)
            ).await;
            counter.fetch_sub(1, Ordering::Relaxed);
        });
    }
    tokio::time::sleep(tokio::time::Duration::from_secs(1)).await;
}
⚠️

المهام المُنشأة يجب أن تكون 'static

المهام الممررة إلى tokio::spawn يجب أن تملك جميع بياناتها (أن تكون 'static). استخدم إغلاقات move و Arc لمشاركة البيانات بين المهام، تمامًا كما مع Threads.

5

Channels الـ Asyncة

يوفر Tokio قنوات مدركة للعمليات الـ Asyncة للتواصل بين المهام. على عكس std::sync::mpsc، هذه Channels تعلّق المهمة (بدلاً من حجب الخيط) عند انتظار الرسائل.

Rust
#[tokio::main]
async fn main() {
    let (tx, mut rx) = tokio::sync::mpsc::channel(100);
    tokio::spawn(async move {
        for i in 0..5 {
            tx.send(i).await.unwrap();
            tokio::time::sleep(
                tokio::time::Duration::from_millis(100)
            ).await;
        }
    });
    while let Some(value) = rx.recv().await {
        println!("Received: {}", value);
    }
}
Rust
// Async producer-consumer pattern
#[tokio::main]
async fn main() {
    let (tx, mut rx) = tokio::sync::mpsc::channel(10);
    // Producer task
    tokio::spawn(async move {
        for i in 0..10 {
            match tx.send(format!("Message {}", i)).await {
                Ok(_) => {},
                Err(_) => break,
            }
            tokio::time::sleep(
                tokio::time::Duration::from_millis(50)
            ).await;
        }
    });
    // Consumer
    let mut count = 0;
    while let Some(msg) = rx.recv().await {
        println!("{}", msg);
        count += 1;
    }
    println!("Processed {} messages", count);
}
💡

Channels المحدودة مقابل غير المحدودة

tokio::sync::mpsc::channel(n) ينشئ قناة محدودة بسعة n. عندما تمتلئ، يتم تعليق send().await حتى تتوفر مساحة. هذا يوفر ضغطًا عكسيًا طبيعيًا لمنع المنتجين من إغراق المستهلكين.

6

ماكرو select! و join!

ماكرو select! في Tokio يسابق بين عقود آجلة متعددة وينفّذ الفرع الذي يكتمل أولاً. join! ينتظر اكتمال جميع Futures بشكل متزامن.

Rust
// select! races multiple futures
#[tokio::main]
async fn main() {
    tokio::select! {
        _ = tokio::time::sleep(
            tokio::time::Duration::from_secs(1)
        ) => {
            println!("Timeout reached");
        }
        result = async_operation() => {
            println!("Operation completed: {}", result);
        }
    }
async fn async_operation() -> String {
    "Done".to_string()
}
Rust
// join! waits for all futures concurrently
#[tokio::main]
async fn main() {
    let result = tokio::join!(
        operation1(),
        operation2(),
        operation3()
    );
    println!("Results: {:?}", result);
    // (1, 2, 3)
async fn operation1() -> i32 { 1 }
async fn operation2() -> i32 { 2 }
async fn operation3() -> i32 { 3 }
✨

select! مقابل join!

استخدم join! عندما تحتاج جميع النتائج (مثل جلب البيانات من واجهات برمجة متعددة). استخدم select! عندما تريد التصرف بناءً على ما يكتمل أولاً (مثل تطبيق مهلات، أو إلغاء عملية بطيئة).

7

Error Handling في الكود الـ Async

Error Handling في الكود الـ Async تتبع نفس Patterns Result وعامل ? كما في Rust المتزامن. عامل ? يعمل داخل async fn تمامًا كما تتوقع.

Rust
async fn potentially_failing_operation() -> Result<String, String> {
    if true {
        Ok("Success".to_string())
    } else {
        Err("Failed".to_string())
    }
#[tokio::main]
async fn main() {
    match potentially_failing_operation().await {
        Ok(value) => println!("Success: {}", value),
        Err(e) => println!("Error: {}", e),
    }
}
Rust
// Using the ? operator in async functions
async fn run_async_operations() -> Result<(), String> {
    let result1 = operation1().await?;
    let result2 = operation2().await?;
    println!("Results: {}, {}", result1, result2);
    Ok(())
async fn operation1() -> Result<i32, String> { Ok(42) }
async fn operation2() -> Result<i32, String> { Ok(43) }
#[tokio::main]
async fn main() {
    match run_async_operations().await {
        Ok(_) => println!("All operations succeeded"),
        Err(e) => println!("Error: {}", e),
    }
}
💡

نشر الأخطاء

عامل ? في Async Functions يعمل بشكل مطابق للكود المتزامن. إذا أرجع عقد آجل منتظَر Err، يتم نشر الخطأ فورًا. ادمجه مع مكتبتي anyhow أو thiserror لمعالجة أخطاء مريحة.

مثال عملي: عميل HTTP غير متزامن

مكتبة reqwest توفر عميل HTTP غير متزامن شائعًا يتكامل بسلاسة مع Tokio. هذا المثال يوضح طلبات HTTP متسلسلة ومتزامنة.

Rust
// Cargo.toml:
// [dependencies]
// reqwest = { version = "0.11", features = ["json"] }
// tokio = { version = "1", features = ["full"] }
#[tokio::main]
async fn main() {
    let client = reqwest::Client::new();
    match client
        .get("https://api.github.com/repos/rust-lang/rust")
        .send()
        .await
    {
        Ok(response) => {
            match response.text().await {
                Ok(text) => println!("Response: {}",
                    &text[0..100.min(text.len())]),
                Err(e) => println!("Failed to read body: {}", e),
            }
        }
        Err(e) => println!("Request failed: {}", e),
    }
}
Rust
// Concurrent HTTP requests
#[tokio::main]
async fn main() {
    let client = reqwest::Client::new();
    let urls = vec![
        "https://example.com",
        "https://example.com/page2",
        "https://example.com/page3",
    ];
    let mut handles = vec![];
    for url in urls {
        let client = client.clone();
        let handle = tokio::spawn(async move {
            match client.get(url).send().await {
                Ok(resp) => println!("Status: {}", resp.status()),
                Err(e) => println!("Error: {}", e),
            }
        });
        handles.push(handle);
    }
    for handle in handles {
        let _ = handle.await;
    }
}
✨

الـ Async مقابل Threads

المهام الـ Asyncة أرخص بكثير من Threads نظام التشغيل. يمكنك إنشاء آلاف المهام الـ Asyncة على مجموعة Threads واحدة. استخدم Async Programming للعمل المرتبط بالإدخال/الإخراج (الشبكة، القرص) وThreads للعمل المرتبط بالـ Handler (الحوسبة، معالجة الصور).
