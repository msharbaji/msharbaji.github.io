Ownership, الميزة الأقوى في Rust

الميزة الأكثر تميزاً في Rust, افهم القواعد التي تضمن Memory Safety بدون Garbage Collector.

1

ما هو الـ Ownership؟

الـ Ownership هو أكثر ميزات Rust تميزاً وهي ضرورية لفهم كيف تضمن Rust Memory Safety بدون Garbage Collector. وهي أيضاً مصدر معظم صعوبات المبتدئين مع Rust. لكن بمجرد فهمها، ستقدّر قوتها.

الـ Ownership هو مجموعة من القواعد التي تحكم كيفية إدارة برنامج Rust للذاكرة. كل قيمة في Rust لها Owner، وعندما يخرج الـ Owner من الـ Scope، يتم إسقاط القيمة (تحرير الـ Memory). هذا يضمن تنظيف الـ Memory بشكل صحيح دائماً.

الـ Stack مقابل الـ Heap

لفهم الـ Ownership، من المفيد فهم كيفية عمل الـ Stack والـ Heap:

  • الـ Stack: يخزن القيم بطريقة الداخل أخيراً يخرج أولاً. جميع البيانات المخزنة على الـ Stack يجب أن يكون لها حجم معروف وثابت. وهو سريع لأنك لا تحتاج للبحث عن مكان لوضع بيانات جديدة؛ فهي تذهب دائماً إلى الأعلى.
  • الـ Heap: تخزن البيانات ذات الحجم غير المعروف في Compile-time أو الحجم الذي قد يتغير. الـ Heap أقل تنظيماً؛ عند وضع بيانات على الـ Heap، تخصص مساحة وتحصل على مؤشر. الوصول إلى Heap data أبطأ من Stack data.
Rust 
fn main() {
    let x = 5;           // Small integer on stack
    let y = "hello";     // String literal (stack)
    let s = String::new();  // String on heap (pointer on stack)
}
2

قواعد Ownership

لدى Rust ثلاث قواعد أساسية للملكية:

  1. كل قيمة في Rust لها Variable يُسمى Ownerها.
  2. لا يمكن أن يكون هناك سوى Owner واحد في وقت واحد.
  3. عندما يخرج الـ Owner من الـ Scope، يتم إسقاط القيمة.
✨

فكرة جوهرية

هذه القواعد الثلاث هي أساس Memory Safety في Rust. يفرضها الـ Compiler في Compile-time، لذلك لا توجد تكلفة أثناء التشغيل لإدارة الـ Memory.

3

Move Semantics

عندما تسند قيمة من Variable إلى آخر، يتم نقل الـ Ownership. يُسمى هذا "نقل":

Rust 
fn main() {
    let s1 = String::from("hello");
    let s2 = s1;  // s1's ownership moves to s2
    // This would cause a compile error:
    // println!("{}", s1);  // Error: s1 has been moved
    println!("{}", s2);  // This works fine
}

لماذا Move Semantics؟

بالنسبة للأنواع البسيطة مثل Integers، تنسخ Rust القيمة بدلاً من نقلها. وذلك لأنها مخزنة على الـ Stack ونسخها رخيص:

Rust 
fn main() {
    let x = 5;
    let y = x;  // x is copied (integers implement Copy)
    println!("x = {}, y = {}", x, y);  // Both are valid
}

Clone: نسخة عميقة صريحة

عندما تريد عمل نسخة من Heap data، استدعِ clone() بشكل صريح:

Rust 
fn main() {
    let s1 = String::from("hello");
    let s2 = s1.clone();  // Explicit deep copy
    println!("s1 = {}, s2 = {}", s1, s2);  // Both are valid
}

سمة Copy

الأنواع التي تُطبّق سمة Copy يتم نسخها تلقائياً بدلاً من نقلها. الأنواع البسيطة مثل Integers والأعداد العشرية وBooleans والأحرف تُطبّق Copy:

Rust 
fn main() {
    let x = 42;        // i32 implements Copy
    let y = x;          // x is copied automatically
    let z: bool = true;  // bool implements Copy
    let w = z;          // z is copied automatically
    println!("x={}, y={}, z={}, w={}", x, y, z, w);
}
4

Functions والـ Ownership

عندما تمرر قيمة إلى Function، يتم نقل الـ Ownership (أو نسخها):

Rust 
fn take_ownership(s: String) {
    println!("I own: {}", s);
}  // s is dropped here
fn copy_integer(x: i32) {
    println!("I have: {}", x);
}  // x is not dropped because i32 implements Copy
fn main() {
    let s = String::from("hello");
    take_ownership(s);  // s's ownership moves into function
    // This would error: println!("{}", s);  // s no longer valid
    let num = 5;
    copy_integer(num);  // num is copied
    println!("num = {}", num);  // num is still valid
}

يمكن للدوال أيضاً إعادة الـ Ownership:

Rust 
fn give_ownership() -> String {
    String::from("hello")
fn take_and_give_back(s: String) -> String {
    s  // Return ownership
fn main() {
    let s1 = give_ownership();
    let s2 = String::from("world");
    let s3 = take_and_give_back(s2);  // s2 moves in, s3 receives ownership
    println!("s1 = {}, s3 = {}", s1, s3);
}
5

الـ References و Borrowing

بدلاً من نقل الـ Ownership، يمكنك إعارة Reference لقيمة. يسمح لك الـ Reference باستخدام قيمة دون أخذ الـ Ownership:

الـ References Immutable

Rust 
fn calculate_length(s: &String) -> usize {
    s.len()
}  // s goes out of scope, but it doesn't own s, so nothing happens
fn main() {
    let s1 = String::from("hello");
    let len = calculate_length(&s1);  // Borrow s1
    println!("'{}' has length {}", s1, len);  // s1 is still valid
}

الـ References الـ Mutable

يمكنك تعديل قيمة من خلال Mutable Reference:

Rust 
fn change_string(s: &mut String) {
    s.push_str(" world");
fn main() {
    let mut s = String::from("hello");
    change_string(&mut s);  // Mutable borrow
    println!("{}", s);  // Prints: hello world
}
⚠️

قواعد Borrowing

في أي وقت، يمكنك الحصول على إما Mutable Reference واحد أو أي عدد من الـ References Immutable. لا يمكنك الحصول على كليهما في نفس الوقت.

Rust 
fn main() {
    let mut s = String::from("hello");
    let r1 = &s;      // Immutable reference
    let r2 = &s;      // Another immutable reference (OK)
    println!("{}, {}", r1, r2);
    // This would error (can't mix mutable and immutable):
    // let r3 = &mut s;
    let r3 = &mut s;  // Mutable reference (now OK after r1, r2 are gone)
    r3.push_str(" world");
    println!("{}", r3);
}
6

نوع الـ Slice

الـ Slice هي Reference لتسلسل متجاور من العناصر داخل مجموعة. Slices لا تأخذ الـ Ownership:

شرائح النصوص

Rust 
fn main() {
    let s = String::from("Hello World");
    let hello = &s[0..5];   // "Hello"
    let world = &s[6..11];  // "World"
    println!("{} {}", hello, world);
}

Array Slices

Rust 
fn main() {
    let a = [1, 2, 3, 4, 5];
    let slice = &a[1..4];  // [2, 3, 4]
    for &item in slice {
        println!("Item: {}", item);
    }
}

Function عملية مع Slices

Rust 
fn first_word(s: &String) -> &str {
    let bytes = s.as_bytes();
    for (i, &item) in bytes.iter().enumerate() {
        if item == b' ' {
            return &s[0..i];
        }
    }
    &s[..]  // Return the whole string
fn main() {
    let s = String::from("hello world");
    let word = first_word(&s);
    println!("First word: {}", word);  // Prints: First word: hello
}
7

Dangling References

تمنع Rust Dangling References, وهي مراجع لبيانات لم تعد موجودة. يكتشفها الـ Compiler في Compile-time:

Rust 
fn dangle() -> &String {  // This won't compile!
    let s = String::from("hello");
    &s  // s is dropped at end of function, so reference is invalid
}  // ERROR: `s` does not live long enough
fn no_dangle() -> String {  // This works!
    let s = String::from("hello");
    s  // Ownership is transferred, not a reference
fn main() {
    let s = no_dangle();
    println!("{}", s);
}
💡

الـ Compiler يحميك

يكتشف الـ Borrow Checker في Rust Dangling References في Compile-time. في C أو C++، ستكون هذه أخطاء Runtime المعروفة بصعوبة تتبعها.

ملخص Ownership

الـ Ownership قوي بمجرد فهمها. إليك مثالاً عملياً يجمع كل شيء معاً:

Rust 
fn main() {
    let mut text = String::from("Ownership");
    // Immutable borrow - we can read
    let len = get_length(&text);
    println!("Length: {}", len);
    // Mutable borrow - we can modify
    add_exclamation(&mut text);
    println!("Text: {}", text);
    // Transfer ownership
    let owned = text;
    println!("Owned: {}", owned);
    // text is no longer valid here
fn get_length(s: &String) -> usize {
    s.len()
fn add_exclamation(s: &mut String) {
    s.push_str("!!!");
}
✨

ورقة Reference للملكية

  • النقل (Move): نقل الـ Ownership من Variable إلى آخر (let s2 = s1;)
  • الاستنساخ (Clone): نسخة عميقة صريحة لHeap data (let s2 = s1.clone();)
  • النسخ (Copy): نسخ تلقائي للأنواع المخزنة على الـ Stack فقط (Integers، Booleans، الأحرف)
  • Borrowing (Borrow): إعارة Reference دون التنازل عن الـ Ownership (&s أو &mut s)
  • الـ Slice (Slice): Reference لجزء من مجموعة (&s[0..5])
