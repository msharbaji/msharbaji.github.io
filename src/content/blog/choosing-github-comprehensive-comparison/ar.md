في [الجزء الأول](/blog/embracing-change-bitbucket-to-github)، تكلمنا عن أسباب ابتعادنا عن BitBucket و Estafette. هالمقال يكمل القصة - نستعرض فيه عملية التقييم اللي وصلتنا لاختيار GitHub.

## معايير الاختيار

قبل ما ندخل في مقارنة المنصات، حددنا إيش اللي يهمنا. المنصة الجديدة لازم تحقق خمس نقاط أساسية:

1. **تكلفة عادلة** - سعر مناسب مقابل الميزات، ويتوسع مع نمونا
2. **أداء أفضل للـ pipelines** - builds أسرع، caching أفضل، syntax مرن
3. **أمان محسّن** - فحص dependencies، إدارة secrets، تحكم بالوصول
4. **صيانة أسهل** - إدارة runners أبسط، أدوات إدارة أفضل
5. **تجربة مطور** - API/CLI قوي، تكامل مع الـ IDE، منظومة مجتمعية

بناءً على واقع السوق، حصرنا التقييم بين **GitHub** و **GitLab**. كلاهما منصة ناضجة ومنتشرة مع قدرات CI/CD قوية. السؤال كان: أيهما يناسب احتياجاتنا أكثر؟

## مقارنة التكلفة

التكلفة كانت عامل رئيسي. مع حوالي 187 مطور و 2.46 مليون دقيقة CI/CD سنوياً، حتى الفروقات الصغيرة في السعر تتراكم بسرعة.

### الأسعار الأساسية

قارنّا GitHub Enterprise مع Advanced Security ضد GitLab Ultimate - وهي الباقات اللي تتضمن ميزات الأمان اللي نحتاجها.

| | GitLab Ultimate | GitHub Enterprise + AS |
|---|---|---|
| SCM لكل مستخدم/سنة | $1,188 | $748.80 |
| إجمالي SCM (187 مستخدم) | $222,156 | $140,000 |
| دقائق CI/CD مجانية/سنة | 600,000 | 600,000 |
| تكلفة الدقيقة الإضافية | $0.01 | $0.008 |
| تكلفة الدقائق الإضافية (1.86M) | $18,600 | $14,880 |

كلا المنصتين تقدم 600,000 دقيقة مجانية. لباقي الـ 1.86 مليون دقيقة، سعر GitHub الأقل للدقيقة وفّر علينا أكثر.

### توقعات ثلاث سنوات مع الخصومات

كلا المزودين عرضوا خصومات للكميات. هذي المقارنة على مدى ثلاث سنوات:

| السنة | GitLab Ultimate | GitHub Enterprise + AS |
|---|---|---|
| السنة الأولى | $155K (خصم 35%) | $101K (خصم 40%) |
| السنة الثانية | $178K (خصم 25%) | $134K (خصم 20%) |
| السنة الثالثة | $202K (خصم 15%) | $151K (خصم 10%) |
| **الإجمالي لـ 3 سنوات** | **$535K** | **$386K** |

GitHub جاء **أوفر بـ $149K** على مدى ثلاث سنوات. وهذا قبل ما نحسب Snyk - GitLab Ultimate يتضمن فحص dependencies أساسي، لكن GitHub Advanced Security يغطي هالجزء، فنوفر $51K إضافية سنوياً على تراخيص Snyk مع GitHub.

### تكاليف التوسع

كمان حسبنا إيش يصير لو زاد استخدامنا للـ CI/CD:

| السيناريو | GitLab Ultimate | GitHub Enterprise + AS |
|---|---|---|
| ضعف الدقائق (4.4M إضافية) | $285K/سنة | $190K/سنة |
| ثلاث أضعاف (6.9M إضافية) | $309K/سنة | $210K/سنة |

سعر GitHub الأقل للدقيقة يعني إن الميزة التكلفية تزيد كل ما توسعنا.

## أداء الـ Pipelines

pipelines أسرع تعني feedback أسرع، وهذا ينعكس مباشرة على إنتاجية المطورين. شغّلنا نفس العمليات على كلا المنصتين وقسنا الفرق.

### المواصفات وسرعة التنفيذ

runners المُستضافة في GitHub تجي بمواصفات أفضل من البداية:

| | GitLab SaaS Runner | GitHub Hosted Runner |
|---|---|---|
| Linux | 1 vCPU, 3.75 GB RAM | 2 vCPU, 7 GB RAM, 14 GB SSD |
| macOS | غير متاح (self-hosted فقط) | 2 vCPU, 14 GB RAM, 14 GB SSD |

فرق المواصفات ظهر في الـ builds الفعلية. لأغلب تطبيقات Java عندنا، GitHub كان أسرع بشكل ملحوظ - أحياناً حتى بدون cache. GitLab كمان يضيف تقريباً دقيقة overhead لكل job بس لتجهيز البيئة ورفع الـ artifacts.

### التخزين المؤقت (Caching)

الـ caching كان فارق واضح:

- **GitLab**: الـ cache يشتغل بس داخل نفس الـ tagged runner. مسح الـ cache يعني مسح كل الـ cache للـ pipeline كامل - ما تقدر تحذف key واحد.
- **GitHub**: الـ cache يشتغل تلقائياً على الـ hosted و self-hosted runners. تقدر تدير cache keys فردية وتشوف حجم كل واحد.

### بنية الـ Pipeline

GitHub Actions يقدم مزايا حقيقية في طريقة تنظيم workflows الـ CI/CD:

- **ملفات YAML متعددة** - كل workflow ملف منفصل بأحداث تشغيل خاصة. في GitLab، كل شي يروح في ملف `.gitlab-ci.yml` واحد، وهذا يصعّب التنقل مع التوسع.
- **مكونات قابلة لإعادة الاستخدام** - GitHub Actions و reusable workflows تسمح للفرق بمشاركة منطق الـ build بدون نسخ YAML.
- **ما فيه تعارضات أسماء** - أسماء الـ jobs والـ steps مُنظمة بشكل صحيح، وتتجنب تعارضات الأسماء اللي تصير في GitLab.
- **تعبيرات غنية** - شروط مفصلة للتحكم بمتى تشتغل الـ jobs والـ steps.

### مشاركة البيانات بين الـ Jobs

كلا المنصتين تدعم artifact upload/download لمشاركة الملفات بين الـ jobs. لكن GitHub عنده كمان **context mechanism** يسمح بتمرير key-value pairs نصية مباشرة بين الـ jobs والـ steps - بدون ما تحتاج تكتب وتقرأ ملفات Dotenv مثل GitLab.

## الأمان

الأمان كان الجزء الأكثر دقة في التقييم. كل منصة عندها نقاط قوة حقيقية.

### وين GitLab يتفوق: الفحص المتكامل

GitLab Ultimate يجي معاه container scanning و dependency scanning و SAST و runtime scanning جاهزة. كمان يوفر security dashboard ويقدر يفحص containerized workloads عبر الـ cluster agent. GitHub يقدم ميزات مشابهة عبر Advanced Security، لكن تحتاج إعداد أكثر عبر Actions.

### وين GitHub يتفوق: إدارة الـ Secrets

GitHub عنده ميزتين مهمتين GitLab ما يوفرها:

- **منع push الـ secrets** - GitHub يفحص كل push عشان يكتشف secrets اللي انضافت بالغلط ويوقفها قبل ما توصل للمستودع. GitLab ما يقدم هالميزة.
- **تكامل مع Google Secret Manager** - GitHub عنده Action رسمي من Google لجلب الـ secrets، بالإضافة لـ Action رسمي من Google لـ Workload Identity Federation (WIF). دعم WIF في GitLab كان لسه في مرحلة alpha وقت التقييم.

### التحكم بوصول الـ Cloud

كلا المنصتين تتعاملان مع وصول الـ cloud والـ cluster، لكن بأساليب مختلفة. GitHub عنده Actions رسمية من Google لـ WIF ووصول GKE cluster. GitLab عنده cluster agent مدمج. ما كان فيه فائز واضح هنا.

## الصيانة وتجربة المطور

### إدارة الـ SCM

كلا المنصتين تغطي الأساسيات، لكن بنقاط قوة مختلفة:

- **GitHub** يوفر أدوار مخصصة وفحوصات حالة فردية على pull requests
- **GitLab** عنده تكامل أفضل مع IdP من البداية

### إدارة الـ Runners

GitLab عنده الأفضلية هنا. الـ runners حقته تندمج بسهولة في GKE clusters والبنية التحتية الموجودة عندنا.

### أدوات المطور

GitHub يفوز في تجربة المطور:

- **تكامل أصلي مع VSCode** مع full-screen mode و timestamp toggle و fine-grained API access
- **GitHub Codespaces** لبيئات تطوير سحابية مؤقتة
- **GitHub CLI** (`gh`) لإدارة PRs والـ issues والـ Actions وأكثر من الترمنال
- **GitHub Copilot** للتطوير بمساعدة الذكاء الاصطناعي

GitLab يوفر web IDE لكن يحتاج تكامل مع طرف ثالث (مثل Gitpod) لبيئات التطوير السحابية.

### المنظومة والمجتمع

GitHub عنده منظومة أكبر بكثير:

- **GitHub Marketplace** بآلاف الـ Actions والـ Apps اللي يدعمها المجتمع والشركات الكبيرة
- **تكاملات واسعة مع أطراف ثالثة** - أغلب أدوات SaaS توفر دعم من الدرجة الأولى لـ GitHub
- **منصة مجتمعية** حيث منظمات متعددة تدعم وتطوّر أدوات مشتركة

## الخلاصة

الانتقال لمنصة CI/CD مُدارة كان القرار الصح لفريقنا. التقييم أكّد إن GitHub Enterprise مع Advanced Security يعطينا أفضل مزيج من كفاءة التكلفة وأداء الـ pipelines وتغطية الأمان وتجربة المطور.

GitLab منصة قوية - الفحص الأمني المتكامل وإدارة الـ runners ممتازين فعلاً. لكن لفريق بحجمنا، مع بنيتنا التحتية واتجاه نمونا، GitHub كان الأنسب على كل الأصعدة.

في المقال القادم، [إدارة GitHub على نطاق المؤسسة باستخدام Terraform](/blog/managing-github-at-scale)، نغطي كيف بنينا منصة خدمة ذاتية لإدارة 400+ مستودع و 27 فريق وسياسات الأمان باستخدام البنية التحتية ككود.
